  <action>
    Wire up the three external services: GitHub (repo), Sanity (CMS project), and Vercel (hosting). This task creates the deployment pipeline.

    **Step 1: Initialize git and create GitHub repo**
    ```
    cd ~/Dropbox/GitHub/la-sprezzatura
    git init
    git config user.name "Paul Albert"
    git config user.email "palbert1@gmail.com"
    git add -A
    git commit -m "feat: scaffold Astro 6 project with Sanity, Tailwind, and Vercel"
    gh repo create la-sprezzatura --private --source=. --push
    ```
    The repo should be created as private (this is a client project). Use `gh` CLI which is already authenticated.

    **Step 2: Create Sanity project**
    Run `npx sanity init` in the project directory with Node 22 active. When prompted:
    - Create a new project named "La Sprezzatura"
    - Use dataset "production"
    - Use the default plan (free)

    Alternatively, if the interactive prompts are problematic, create the project via `npx sanity init --create-project "La Sprezzatura" --dataset production` or use the Sanity dashboard at sanity.io/manage.

    Capture the project ID from the output. It will look like a random alphanumeric string (e.g., `abc123de`).

    **Step 3: Update .env with real Sanity project ID**
    Replace `placeholder` in .env with the actual project ID:
    ```
    PUBLIC_SANITY_PROJECT_ID=<actual_project_id>
    PUBLIC_SANITY_DATASET=production
    ```

    **Step 4: Configure Sanity CORS origins**
    Add CORS origins for the Sanity project so the Studio and data fetching work:
    ```
    npx sanity cors add http://localhost:4321 --credentials
    npx sanity cors add https://la-sprezzatura.vercel.app --credentials
    ```
    If the Vercel URL differs from `la-sprezzatura.vercel.app`, use the actual URL from Step 5.

    **Step 5: Import project to Vercel**
    Use the Vercel dashboard (vercel.com/new) to import the GitHub repo:
    1. Go to vercel.com/new
    2. Import the `la-sprezzatura` repository from GitHub
    3. Framework preset should auto-detect as Astro
    4. Add environment variables in Vercel project settings:
       - `PUBLIC_SANITY_PROJECT_ID` = the actual project ID
       - `PUBLIC_SANITY_DATASET` = production
    5. Deploy

    If Vercel CLI is preferred: `npm i -g vercel && vercel --yes` then `vercel env add PUBLIC_SANITY_PROJECT_ID` and `vercel env add PUBLIC_SANITY_DATASET`.

    NOTE: Verify the Vercel account is on the Pro plan (required for commercial sites per Vercel ToS). If on Hobby, the user must upgrade before the site goes live in Phase 4. For staging purposes during development, Hobby may be acceptable.

    **Step 6: Verify deploy triggered**
    After Vercel import, a deploy should trigger automatically. Wait for it to complete. Note the preview URL (e.g., `https://la-sprezzatura.vercel.app`).

    **Step 7: Update Sanity CORS if needed**
    If the Vercel URL is different from what was added in Step 4, add the correct URL:
    ```
    npx sanity cors add https://<actual-vercel-url> --credentials
    ```

    **Step 8: Commit updated .env values and push**
    Do NOT commit .env (it is in .gitignore). But do push any other changes:
    ```
    git add -A
    git commit -m "chore: configure Sanity project and Vercel deployment"
    git push
    ```
    This push should trigger a new Vercel deploy with the correct Sanity project ID.

    **Step 9: Verify Wix site is unaffected**
    Run `curl -sI https://lasprezz.com | head -5` to confirm the Wix site is still serving. The response should NOT reference Vercel. This is a safety check -- no DNS changes were made, so it should pass trivially.
