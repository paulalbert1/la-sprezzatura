# Phase 46.1 Plan 09: Round-4 Gap Closure Summary

**One-liner:** Round-4 fix for CR-R3-01 BLOCKER (dead className-hook palette) + 6 round-2 WR carryovers + 3 round-3 quality items, landed in a single atomic commit at `b75ab0b`.

## What Shipped

Round-3 code review (`a59cf25` in `46.1-REVIEW.md`) flagged a BLOCKER: the dark-mode lock palette in `EmailShell.tsx` targeted className hooks (`pill-*`, `cta-*`, `text-stone`, `text-charcoal`, `border-cream-dark`) that **never appeared on rendered elements**. `@react-email/tailwind` compiles utility classes (`bg-cream`, `text-stone`, etc.) to inline styles and strips the className before HTML reaches the wire; literal pill/cta classes were never applied as className= props in source code. Result: gap-4 (Outlook desktop dark-mode contrast inversion) was unfixed despite 46.1-04's lock landing — only the body-bg paint via `<table bgcolor>` + inline `<body style>` worked.

This plan ships the round-4 fix as a single atomic commit per D-20:

1. **D-15 className binding** — `Procurement.tsx` pill `<span>` now carries `className={\`pill-${it.status}\`}` alongside its existing inline style; `EmailButton.tsx` `<Button>` now carries `className={\`cta-${variant}\`}`. Literal class strings are NOT Tailwind utilities, so they pass through the compiler unchanged. The HARD verification gate at task-1 PASSED: snapshot regen + grep returned 4 `class="pill-*"` matches and 5 `class="cta-*"` matches.
2. **D-16 dead-rule deletion** — 7 `[data-ogsc] .text-*` rules + 1 `.border-cream-dark` rule deleted from BOTH the `[data-ogsc]/[data-ogsb]` block AND the `@media (prefers-color-scheme: dark)` block. Knock-on: WorkOrder's `<Hr className="border-cream-dark">` is now intentionally unprotected at the class layer (the Tailwind compile path was already painting it `#eaeaea`, not `#E8DDD0`); round-5 visual UAT is the gate.
3. **D-17 catch-all deletion** — `[data-ogsc] * { color: inherit !important }` deleted. With D-15 binding pill className hooks, per-class `!important` rules now re-pin colors after auto-darken instead of being cancelled by the catch-all.
4. **D-18 #3 CSS-gen refactor** — module-load-time generation via `buildPillRules()` reading `STATUS_PILL_STYLES`. Eliminates ~40 lines of hand-mirrored CSS; drift between admin UI palette + email render + dark-mode lock is now structurally impossible.
5. **D-18 #1+#2 test rewrites** — `EmailShell.test.ts` rewritten from hex-substring shape to className-binding shape (63 tests, was 29). `SendUpdate.test.ts` width + valign tests rewritten from magic-number-9 floor to per-row invariant (4 tests, decoupled from fixture row count).
6. **D-19 round-2 WR carryovers** — WR-01/WR-02 (`stripLeadingGreeting` takes `clientFirstName`, case-insensitive exact-match), WR-03 (`MAX_LEN` exported + textarea `maxLength` + character counter), WR-05 (MSO conditional out of `<div>` in `<head>`), WR-06 (pill `borderRadius` deleted).
