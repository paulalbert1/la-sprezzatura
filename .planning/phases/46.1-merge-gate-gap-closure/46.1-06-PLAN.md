**Step 2 -- Identify the four candidate techniques the spike must evaluate (per D-10).**

The four candidates are pre-locked by D-10 (the spike documents them; it does NOT pick a different four):

(a) **`<table bgcolor>` body wrapper** -- wrap the `<Body>` content in a `<table bgcolor="#FAF8F5">` (or equivalent top-level table) so the cream paint comes from a table-level HTML attribute (NOT a CSS rule), which Outlook-for-Mac's auto-darken pipeline historically respects more reliably than `<style>`-block CSS.
(b) **`@media (prefers-color-scheme: dark)` block** -- a media query inside the existing `<style>` block, with explicit overrides matching the locked light palette. Some Outlook variants honor this even when ignoring `color-scheme: only light`.
(c) **Class-hook selectors with `!important`** -- e.g., `body[class~="email-shell"]` or `.email-shell-locked` selectors that match where `[data-ogsc]` doesn't. The existing 46.1-04 lock uses `[data-ogsc]`/`[data-ogsb]` attribute selectors; this candidate adds a class-based selector path as a third match surface.
(d) **Inline body-attribute paint** -- `<body style="background:#FAF8F5">` and on key wrappers, instead of relying on CSS class -> compiled-style. Fallback for the case where Outlook-for-Mac strips `<style>` blocks in `<head>`. Inline `style=` attributes are harder for Outlook to override because they have higher specificity than auto-darken's injected rules.

For EACH candidate (a)-(d), the spike documents:
- **What it fixes:** the specific Outlook-for-Mac failure mode this technique targets
- **Variants honored:** which of {New Outlook for Mac dark, Outlook for Mac light, Outlook for Windows classic dark/light, Outlook web M365 dark, Apple Mail dark} the technique survives in (per cited sources)
- **Known landmines:** failure modes the technique introduces (e.g., (a) `<table bgcolor>` doesn't survive when Outlook auto-darken paints body color higher up; (c) class-hook selectors don't survive if Outlook-Mac strips `<style>` entirely)
- **Source URLs:** at least one cited reference per candidate
- **Confidence rating:** `high` / `medium` / `low` based on independent corroboration in 2+ sources

If during the literature review the executor encounters a 5th or 6th additional technique not listed in (a)-(d), document it under "Additional candidates surfaced during research" but DO NOT replace the locked four -- the spike must evaluate all four pre-locked candidates, not a different set.
