## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, tight text spacing |
| sm | 8px | Compact element spacing, pill/badge internal padding |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, card padding (p-6 = 24px) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Section breaks between milestones/procurement/artifacts sections (py-12) |
| 3xl | 64px | Page top/bottom padding on desktop (py-16 to py-20 range) |

Exceptions:
- Touch targets for approval/action buttons: minimum 44px height (accessibility requirement)
- Progress bar height: 4px (slim decorative bar -- one spacing unit)
- Timeline connector dots: 28px diameter (7 x 4, matches existing MilestoneTimeline.astro pattern of `w-7 h-7`)

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (text-sm) | 400 (normal) | 1.7 | DM Sans | Paragraph text, table cells, notes content |
| Label | 12px (text-xs) | 400 (normal) | 1.5 | DM Sans | Uppercase tracking-widest labels, status category headers, metadata dates, confidentiality notice |
| Heading | 20px (text-xl) | 300 (light) | 1.2 | Cormorant Garamond | Section headings (Milestones, Procurement, Artifacts), artifact names |
| Display | 32px mobile / 40px desktop (text-3xl md:text-4xl) | 300 (light) | 1.2 | Cormorant Garamond | Project title in header |

Weights used: 300 (light, headings) and 400 (normal, body). Only two weights are permitted. Emphasis for current stage labels and client names in notes is achieved via `text-charcoal` color contrast against `text-stone` surroundings and `tracking-wide` letter-spacing -- NOT via a third font weight.

Source: Existing patterns in `dashboard.astro` (text-4xl md:text-5xl for page heading, text-xl for cards), `SectionHeading.astro`, `LoginForm.tsx`

## Interaction States

### Milestone Item States

| State | Visual Treatment |
|-------|-----------------|
| Complete | Solid terracotta dot + checkmark SVG + stone text (not muted) + date in stone |
| Current (next upcoming) | Terracotta ring dot (ring-4 ring-terracotta/20) + charcoal text + date in charcoal |
| Upcoming | Cream-dark dot with stone-light border + stone text + date with relative indicator ("in 4 days") |
| Overdue (past date, incomplete) | Stone-light border dot + terracotta-light text for date + subtle amber/stone background tint on the date badge |

### Procurement Status Badges

| Status | Badge Style |
|--------|-------------|
| Ordered | `bg-terracotta/10 text-terracotta` (active) |
| Warehouse | `bg-terracotta/10 text-terracotta` (active) |
| In Transit | `bg-terracotta/10 text-terracotta` (active) |
| Pending | `bg-stone-light/20 text-stone` (neutral) |
| Delivered | `bg-emerald-50 text-emerald-700` (positive) |
| Installed | `bg-emerald-50 text-emerald-700` (positive) |

Badge shape: `px-3 py-1 rounded-full text-xs font-body tracking-wide` (matches StatusBadge.astro pattern)

### Artifact Card States

| State | Visual Treatment |
|-------|-----------------|
| Pending review | Terracotta left border accent (border-l-4 border-terracotta) + "Awaiting Your Review" label + approval buttons visible |
| Approved | Emerald badge "Approved" + approval buttons hidden + approved version prominent |
| Changes requested | Stone badge "Changes Requested" + feedback text visible + approval buttons hidden until new version uploaded |
| Contract (signed) | "Signed" emerald badge + signed PDF as primary download |
| Previous version | opacity-50 + smaller text + collapsed by default |

### Artifact Approval Flow

1. Client clicks "Approve Version" button
2. Confirmation dialog appears:
   - Checkbox: "I confirm this approval on behalf of all parties associated with this project." (AUTH-04, fixed wording)
   - "Confirm Approval" button (disabled until checkbox checked)
   - "Keep Reviewing" link (dismisses dialog, returns to artifact card)
3. On confirm: action submits, decision log entry created, button state transitions to "Approved"

1. Client clicks "Request Changes" button
2. Dialog appears:
   - Required textarea: "What changes would you like?" (min 1 character)
   - "Submit Feedback" button (disabled until text entered)
   - "Go Back" link (dismisses dialog, returns to artifact card)
3. On confirm: action submits, decision log entry created with feedback text, status transitions to "Changes Requested"

### Form States (all React forms: ArtifactApprovalForm, ClientNoteForm, WarrantyClaimForm)

| State | Visual Treatment |
|-------|-----------------|
| Idle | Default form appearance, submit button terracotta |
| Submitting | Button text changes to "Submitting..." + `disabled:opacity-60 disabled:cursor-not-allowed` |
| Success | Form collapses to success message: "Note submitted" / "Approval recorded" in stone text, fades after 3 seconds |
| Error | Error banner: `border border-terracotta/30 bg-terracotta/5 px-5 py-4 text-sm text-charcoal` (established LoginForm.tsx pattern) |

### Completed Project State

| Element | Treatment |
|---------|-----------|
| PostProjectBanner | Full-width cream-dark banner: "Project Complete" heading + completion date + "This project was completed on [date]. You can view all details below." |
| Action buttons | All hidden (Approve Version, Request Changes, Add Note, Submit Warranty) |
| Interactive forms | Not rendered |
| Content | Fully readable, not muted (client needs to reference milestones, procurement, artifacts) |
| Card hover states | Disabled (no pointer cursor) |

Exception: Reopened projects (for warranty) re-enable the WarrantyClaimForm section only.

### Loading and Error States

| Scenario | Treatment |
|----------|-----------|
| Page loading | Server-rendered (SSR) -- no client-side loading spinner needed. Astro renders full page on server. |
| Action submitting | Button disabled + "Submitting..." text (per form states above) |
| Action error | Error banner pattern from LoginForm.tsx |
| Empty milestones | Section heading visible + "No milestones added yet. Liz will update this as your project progresses." |
| Empty procurement | Section heading visible + "No procurement items added yet. Items will appear here as they're ordered." |
| Empty artifacts | Section heading visible + "No documents uploaded yet. Liz will share design materials here as your project progresses." |
| Project not found | Redirect to dashboard with error toast or 404 page matching portal styling |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus visible | `outline: 2px solid var(--color-terracotta); outline-offset: 3px` (established in global.css) |
| Form labels | All inputs have associated `<label>` elements with `for`/`htmlFor` attributes |
| Error announcements | Error banners use `role="alert"` (established LoginForm.tsx pattern) |
| Button states | `disabled:opacity-60 disabled:cursor-not-allowed` + `aria-disabled` |
| Semantic headings | h1 (project title) > h2 (section headings) > h3 (individual items) |
| Table accessibility | `<table>` with `<thead>`, `<th scope="col">`, proper `<caption>` for procurement |
| Collapsible sections | `<details>/<summary>` or `aria-expanded` + `aria-controls` for version history and decision log |
| Status conveyed beyond color | All status badges include text labels, not color alone |
| Touch targets | Minimum 44px height on all interactive elements (buttons, links, form controls) |

## Procurement Table Specification

### Columns

| Column | Width | Alignment | Mobile |
|--------|-------|-----------|--------|
| Item | auto (flexible) | left | Always visible |
| Status | 120px | center | Always visible |
| Install Date | 100px | left | Hidden on mobile, shown on hover/expand |
| MSRP | 100px | right | Always visible |
| Savings | 100px | right | Always visible |
| Tracking | 100px | left | Hidden on mobile, shown on hover/expand |

### Sorting
Default sort: by status priority (Pending > Ordered > Warehouse > In Transit > Delivered > Installed). No interactive sorting controls -- the volume of items per project is small enough (<50 items per RESEARCH.md) that a fixed logical sort is sufficient.

### Tracking Links
Tracking numbers render as clickable links. Display: carrier icon (text abbreviation: "UPS", "FedEx", "USPS") + truncated number. Links open in new tab (`target="_blank" rel="noopener"`).
