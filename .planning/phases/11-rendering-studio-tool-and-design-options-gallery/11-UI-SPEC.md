# Phase 11 -- UI Design Contract

> Visual and interaction contract for the Sanity Studio rendering tool (wizard, chat view, session list, Design Options tab) and the client portal Design Options gallery section (favorites, comments, lightbox). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase has two distinct rendering contexts with separate design systems:
1. **Sanity Studio tool** -- `@sanity/ui` primitives, Studio's built-in theme (dark/light mode aware)
2. **Client portal gallery** -- Tailwind CSS 4, project's cream/terracotta/charcoal design system

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn) |
| Preset | not applicable |
| Component library (Studio) | `@sanity/ui` 3.1.13 (Card, Button, Stack, Dialog, Tab, TabList, TextInput, TextArea, Select, Badge, Flex, Grid, Spinner, Label, Text) |
| Component library (Portal) | none (custom Astro + React components with Tailwind CSS 4) |
| Icon library (Studio) | `@sanity/icons` (SparklesIcon, ImageIcon, StarFilledIcon, StarIcon, HeartIcon, DownloadIcon, EllipsisHorizontalIcon, ArrowLeftIcon, CloseIcon, UploadIcon, SearchIcon, PaperclipIcon, CheckmarkCircleIcon, WarningOutlineIcon, ErrorOutlineIcon, DocumentIcon) |
| Icon library (Portal) | Inline SVG (established pattern -- heart outline/filled, comment bubble, close X, nav arrows) |
| Font (Portal) | Cormorant Garamond (headings, `--font-heading`) + DM Sans (body, `--font-body`) |
| Font (Studio) | Inherited from `@sanity/ui` theme (system font stack) |

Source: Established Phase 2, continued through Phases 5-10. Studio uses `@sanity/ui` exclusively (no Tailwind inside Studio tool). Portal uses project Tailwind system.

## Spacing Scale

### Portal Components (Tailwind)

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, heart icon padding |
| sm | 8px | Gallery card gap between image and caption, comment item spacing |
| md | 16px | Gallery grid gap, lightbox content padding, comment input spacing |
| lg | 24px | Section heading to grid gap (mb-6), confidentiality banner padding, lightbox caption area |
| xl | 32px | Gallery section vertical padding (py-8), lightbox side padding |
| 2xl | 48px | Section breaks (py-12), lightbox vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- 12px (`gap-3`, `space-y-3`): Compact list spacing for comment thread items. Established codebase pattern. Multiples-of-4 compliant (12 = 3 x 4).
- 44px minimum touch target on heart toggle button and lightbox navigation arrows -- WCAG 2.2 Target Size.

### Studio Components (`@sanity/ui`)

Studio components use `@sanity/ui` space tokens (space={1} through space={5}, mapping to 4px increments). Specific values:

| `@sanity/ui` space | Value | Usage |
|---------------------|-------|-------|
| space={1} | 4px | Inline element gaps, badge margins |
| space={2} | 8px | Compact card internal gaps, stepper step spacing |
| space={3} | 12px | Form field spacing, wizard step content padding |
| space={4} | 16px | Card padding, stack spacing, dialog content gaps |
| space={5} | 20px | Major section breaks within tool |

Studio spacing follows `@sanity/ui` conventions, not project Tailwind tokens.

| `default` | Session cards, wizard cards, chat messages |

## Sanity Studio: Rendering Tool

### Tool Registration

```
sanity.config.ts
  tools: [renderingTool()]
    name: "rendering"
    title: "Rendering"
    icon: SparklesIcon
    component: RenderingTool
```

Appears as a sidebar icon (sparkles) alongside Structure and Vision in Sanity Studio.

### Main Tool Layout (RenderingTool)

```
Card (height="fill", overflow="auto")

  [Tab bar -- visible when no session/wizard active]
    TabList (space={1})
      Tab: "Sessions" (default active)
      Tab: "Design Options"

  [Content area -- switches based on state]
    activeTab === "sessions" && !activeSession && !showWizard:
      --> SessionList

      --> DesignOptionsTab

      --> WizardContainer

      --> ChatView (with back button to return to list)
```

State management: React context + useReducer for tool-level state (activeTab, activeSessionId, showWizard, wizardData). Component-local useState for UI state (form fields, loading).

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| Mobile (<640px) | Gallery: 1 column. Lightbox: full-width image, comment panel below. Heart + caption stacked. |

Studio tool renders inside Sanity Studio's work area which has its own responsive behavior. The tool should:
- Session list: single column, full-width cards (Studio work area is typically 60-80% of viewport)
- Wizard: single column form layout, full-width fields
- Chat view: full-width thread, messages max-width 75-85%
- Design Options tab: responsive grid via `@sanity/ui` Grid (columns={[1, 2, 3]})
- Thumbnail strip: horizontal scroll with `overflow-x: auto`
