**1. Create `src/sanity/components/rendering/Wizard/WizardContainer.tsx`:**

4-step wizard with stepper bar and navigation. Uses `useToolContext()` for wizardData state and dispatch.

State management:
- `currentStep`: 1-4 (local useState)
- `isGenerating`: boolean (local)
- `generationError`: string | null (local)

Stepper bar (top): 4 circles with labels. Active = primary background + white number. Completed = positive background + checkmark. Future = muted border + muted number. Connector lines between. Completed steps are clickable (navigate back preserving data).

Step rendering: switch on currentStep to render StepSetup/StepUpload/StepClassify/StepDescribe.

Navigation buttons (bottom):
- Step 1: "Discard Session" (ghost, left) -- shows abandon confirmation dialog if `isWizardDirty(wizardData)`, else dispatches CLOSE_WIZARD
- Steps 2-4: "Back" (ghost, left)
- Steps 1-3: "Next" (primary, right) -- disabled if required fields missing (Step 1: title required)
- Step 4: "Generate" (primary, SparklesIcon, right) -- disabled if description empty

Abandon confirmation dialog (per UI spec lines 489-501):
- `@sanity/ui` Dialog with header "Discard Session?"
- Body: "Your uploads and settings will be lost."
- "Keep Editing" button (ghost) + "Discard" button (critical)

Generate flow (Step 4):
1. Set `isGenerating = true`
2. POST `/api/rendering/generate` with wizardData fields + sanityUserId from context
3. Receive `{ sessionId }` from 202 response
4. Poll `/api/rendering/status?sessionId=xxx` every 2 seconds
5. On `status === "complete"`: dispatch CLOSE_WIZARD, then dispatch OPEN_SESSION with sessionId, call refreshUsage()
6. On `status === "error"`: set generationError, set isGenerating = false, re-enable Generate
7. During generation: show GeneratingOverlay, all inputs disabled

Cancel generation: AbortController on the polling. Set isGenerating = false, re-enable inputs.
