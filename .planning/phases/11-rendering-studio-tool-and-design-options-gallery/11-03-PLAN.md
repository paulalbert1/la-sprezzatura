Props: `sessionId: string`

Layout (per UI spec lines 503-579):
```
Card (height="fill", display="flex", style={{ flexDirection: "column" }})

  [Header bar]
    Flex (align="center", padding={3}, style={{ borderBottom: "1px solid var(--card-border-color)" }})
      Button (icon=ArrowLeftIcon, mode="ghost") -- dispatch BACK_TO_LIST
      Stack (space={1}, style={{ flex: 1, paddingLeft: 12, paddingRight: 12 }})
        Text (size={2}, weight="semibold"): session.sessionTitle
        Text (size={1}, muted): session.project?.title || "Scratchpad"
      UsageBadge (compact variant)

  [Thumbnail strip]
    ThumbnailStrip (renderings, activeIndex, onSelect)

  [Chat thread -- scrollable]
    Stack (space={4}, padding={4}, style={{ flex: 1, overflowY: "auto" }})
      Build conversation from session data:
      - Group conversation entries and renderings chronologically
      - User messages: ChatMessage with role="user"
      - Model responses: ChatMessage with role="model" + RenderingCard for associated rendering
      - Generating skeleton: shown when session.status === "generating"
      - Error renderings: error card variant of RenderingCard

  [Input bar -- bottom]
    Flex (align="center", gap={2}, padding={3}, style={{ borderTop: "1px solid var(--card-border-color)" }})
      Button (icon=PaperclipIcon, mode="ghost", title="Attach image")
        onClick: open file picker for mid-session image upload
      TextInput (style={{ flex: 1 }}, placeholder="Describe refinements...")
      Button (icon=SparklesIcon, tone="primary", title="Refine")
        Disabled when: text empty AND no new images, or isRefining, or usage limit reached
```

State management:
- `session`: fetched via GROQ `RENDERING_SESSION_BY_ID_QUERY` using `useClient()` on mount
- `isRefining`: boolean, true while waiting for refinement to complete
- `refinementText`: string (text input value)
- `attachedImages`: WizardImage[] (mid-session uploads)
- `showAttachCard`: boolean (shows quick-classify card above input)

Refine flow:
1. POST `/api/rendering/refine` with `{ sessionId, refinementText, newImages: attachedImages, sanityUserId }`
2. Set isRefining = true, clear input, show skeleton
3. Poll `/api/rendering/status` every 2 seconds
4. On complete: re-fetch session, show new rendering in thread, call refreshUsage()
5. On error: show error card in thread

Mid-session image upload (per UI spec lines 563-579):
- Paperclip click opens file picker (same accept types as wizard)
- Upload to Blob, then show quick-classify card above input bar
- Card: thumbnail (48x48) + type dropdown + placement input + remove button
- Attached image sent with refinement text on Refine click

Auto-scroll: scroll chat thread to bottom when new messages appear or when generating skeleton shows.

Session data refresh: re-fetch full session after generation completes.
