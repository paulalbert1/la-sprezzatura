### Pre-Transfer: Cloudflare Account Setup

1. Log in to Cloudflare (or create account at https://dash.cloudflare.com)
2. Ensure payment method is on file (required for domain transfers)

### Domain Group A: lasprezzny.com and casadeolivier.com (GoDaddy with Full Protection)

**For EACH domain (do one at a time, verify before starting the next):**

1. **Add domain to Cloudflare DNS** (before transfer):
   - Cloudflare Dashboard > Add a Site > enter domain name
   - Select Free plan
   - Cloudflare will scan existing DNS records -- review and confirm they match DNS-AUDIT.md
   - Cloudflare provides two nameservers (e.g., ada.ns.cloudflare.com, greg.ns.cloudflare.com)

2. **Prepare at GoDaddy:**
   - Log in to GoDaddy > My Products > Domains
   - Select the domain > Domain Settings
   - Turn OFF "Domain Protection" (Full Protection plan)
   - Turn OFF "Domain Lock" (transfer lock)
   - Check for DNSSEC: if enabled, remove DS records FIRST and wait 24 hours before proceeding
   - Get EPP Authorization Code: Domain Settings > Transfer domain away > Get authorization code
   - Note: EPP codes expire in 5-14 days -- initiate transfer the same day

3. **Change nameservers to Cloudflare at GoDaddy:**
   - Domain Settings > Nameservers > Change > Custom
   - Enter the two Cloudflare nameservers from step 1
   - Save
   - Wait for Cloudflare dashboard to show zone as "Active" (usually 5-30 minutes, can take up to 24 hours)

4. **Initiate transfer at Cloudflare:**
   - Cloudflare Dashboard > Domain Registration > Transfer Domains
   - Enter the domain name
   - Paste the EPP authorization code from GoDaddy
   - Confirm the transfer (Cloudflare charges at-cost: ~$10.44/year for .com)
   - GoDaddy will send a confirmation email -- approve the transfer
   - Transfer typically completes in 1-5 days

5. **Verify after zone is Active (before transfer completes):**
   ```bash
   dig NS {domain} +short  # Should show Cloudflare nameservers
   ```

### Domain Group B: lasprezzaturany.com (GoDaddy, already using Cloudflare nameservers)

1. This domain is already using Cloudflare nameservers -- the DNS zone should already be Active in Cloudflare
2. If NOT in Cloudflare: Add it as a site (same as step 1 above)
3. At GoDaddy: Turn OFF Domain Lock, get EPP authorization code
4. At Cloudflare: Initiate domain registration transfer with EPP code
5. Verify: `dig NS lasprezzaturany.com +short` returns Cloudflare nameservers

### Domain Group C: lasprezz.com (Wix -- requires intermediate registrar)

**CRITICAL: Wix does not allow nameserver changes while domain is registered with them. lasprezz.com requires a two-step transfer process.**

1. **Transfer from Wix to GoDaddy (intermediate registrar):**
   - Log in to Wix > Settings > Domains > lasprezz.com
   - Click "Transfer Away" or "Get Transfer Code"
   - Wix will send an EPP authorization code to the domain contact email
   - At GoDaddy: Domains > Transfer > enter lasprezz.com and the EPP code
   - Approve the transfer (Wix may send a confirmation email)
   - Wait for transfer to complete (1-5 days)

2. **At GoDaddy (after transfer arrives):**
   - Change nameservers to Cloudflare (same as Domain Group A step 3)
   - Wait for Cloudflare zone to show "Active"

3. **IMPORTANT: 60-day ICANN lock applies.**
   - After lasprezz.com transfers to GoDaddy, there is a mandatory 60-day wait before it can transfer again
   - The site and email will work perfectly fine during this period -- nameservers point to Cloudflare
   - After 60 days: unlock at GoDaddy, get new EPP code, transfer registration to Cloudflare
   - This does NOT block go-live

4. **Do NOT cancel the Wix subscription before the domain transfer completes.** Wix may complicate the transfer if the subscription is canceled first.

### Step 1: Add lasprezz.com as Custom Domain in Vercel

1. Go to Vercel Dashboard > la-sprezzatura project > Settings > Domains
2. Click "Add" and enter: lasprezz.com
3. Vercel will suggest adding www.lasprezz.com as a redirect -- accept this
4. Note: Vercel will show "Invalid Configuration" until DNS records are added -- this is expected

### Step 2: Configure DNS Records in Cloudflare

Go to Cloudflare Dashboard > lasprezz.com > DNS > Records

**IMPORTANT: Remove all existing records first** (especially any Wix A records, Google MX records, or stale TXT records). Reference DNS-AUDIT.md for what was there.

**Add these records (order does not matter):**

**Site: Vercel hosting (CNAME flattening for apex)**
| Type | Name | Content | Proxy | TTL |
|------|------|---------|-------|-----|
| CNAME | @ | cname.vercel-dns.com | DNS only (gray cloud) | 5 min |
| CNAME | www | cname.vercel-dns.com | DNS only (gray cloud) | 5 min |

**CRITICAL: Proxy MUST be OFF (DNS only / gray cloud) for Vercel records. Orange cloud breaks Vercel SSL.**

**Email: Microsoft 365**
| Type | Name | Content | Priority | Proxy | TTL |
|------|------|---------|----------|-------|-----|
| MX | @ | {tenant-specific}.mail.protection.outlook.com | 0 | N/A | 5 min |
| CNAME | autodiscover | autodiscover.outlook.com | N/A | DNS only | 5 min |

Note: The exact MX value comes from M365 admin center > Settings > Domains > lasprezz.com > DNS records. It will be in the format: `lasprezz-com.mail.protection.outlook.com` or similar.

**Email Authentication: SPF (SINGLE record combining M365 + Resend)**
| Type | Name | Content | TTL |
|------|------|---------|-----|
| TXT | @ | v=spf1 include:spf.protection.outlook.com include:amazonses.com -all | 5 min |

**CRITICAL: Only ONE SPF TXT record. Multiple SPF records cause permerror and email delivery failure.**

Note: Verify the Resend SPF include value from the Resend dashboard (likely `include:amazonses.com` but confirm).

**Email Authentication: DKIM for M365 (values from Plan 12-01 Task 2 Step 5)**
| Type | Name | Content | Proxy | TTL |
|------|------|---------|-------|-----|
| CNAME | selector1._domainkey | selector1-lasprezz-com._domainkey.{tenant}.onmicrosoft.com | DNS only | 5 min |
| CNAME | selector2._domainkey | selector2-lasprezz-com._domainkey.{tenant}.onmicrosoft.com | DNS only | 5 min |

Replace `{tenant}` with the actual tenant name from M365 admin.

**Email Authentication: DMARC (monitoring mode)**
| Type | Name | Content | TTL |
|------|------|---------|-----|
| TXT | _dmarc | v=DMARC1; p=none; rua=mailto:paul@lasprezz.com | 5 min |

**Transactional Email: Resend (send.lasprezz.com)**
1. Go to Resend Dashboard > Domains > Add Domain > enter: send.lasprezz.com
2. Resend will provide DNS records (typically TXT for SPF, MX, and CNAME for DKIM)
3. Add all Resend-provided records to Cloudflare for the send.lasprezz.com subdomain
4. Back in Resend dashboard: click "Verify" -- should show "Verified" within minutes

**M365 Domain Verification (if not already done in Plan 12-01):**

|------|------|---------|-----|
| TXT | @ | MS=ms{XXXXXXXX} | 5 min |

(Value from M365 admin center. Can be removed after domain is verified.)

### Step 3: Configure Cloudflare SSL/TLS

1. Cloudflare Dashboard > lasprezz.com > SSL/TLS > Overview
2. Set encryption mode to: **Full (Strict)**
3. Edge Certificates tab:
   - Always Use HTTPS: ON
   - Automatic HTTPS Rewrites: ON
   - Minimum TLS Version: 1.2
   - HSTS: Enable with max-age 6 months, includeSubDomains ON, preload ON

### Step 4: Verify Vercel SSL

1. Go to Vercel Dashboard > la-sprezzatura project > Settings > Domains
2. lasprezz.com should show a green checkmark and "Valid Configuration"
3. If it shows "Awaiting DNS" or "Invalid Configuration": wait 5-15 minutes for propagation
4. If SSL cert fails: confirm Cloudflare proxy is OFF (DNS only) for the CNAME records
