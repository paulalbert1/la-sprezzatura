1. NS records: `dig NS {domain} +short`
2. A records: `dig A {domain} +short` and `dig A www.{domain} +short`
3. MX records: `dig MX {domain} +short`
4. TXT records (SPF, DKIM, DMARC, verification): `dig TXT {domain} +short`
5. CNAME records: `dig CNAME www.{domain} +short`, `dig CNAME autodiscover.{domain} +short`
6. DKIM selectors: `dig CNAME selector1._domainkey.{domain} +short`, `dig CNAME selector2._domainkey.{domain} +short`
7. DMARC: `dig TXT _dmarc.{domain} +short`
8. Registrar: `whois {domain} | grep -i registrar` (first 5 lines)

Human performs M365 migration from GoDaddy-managed to Microsoft-direct. DNS audit is complete. Now the M365 email must be migrated before any DNS changes (per locked decision).

Follow this step-by-step M365 migration runbook. Complete each step and verify before proceeding.

### Step 1: Attempt GoDaddy Tenant Handoff (Simplest Path)

1. Log in to GoDaddy admin panel
2. Navigate to "My Products" > "Microsoft 365"
3. Look for "Transfer to Microsoft" or "Disconnect from GoDaddy" option
4. If available: Follow the handoff wizard -- GoDaddy relinquishes admin control of the existing tenant
5. If NOT available: Call GoDaddy US support (480-505-8877) and request a "tenant handoff" or "tenant release" for the M365 subscription
6. **If handoff succeeds:** Skip to Step 3 (no data migration needed -- existing data stays in place)
7. **If handoff unavailable:** Proceed to Step 2 (new tenant + migration)

### Step 2: New Microsoft Direct Tenant (Fallback)

Only if GoDaddy tenant handoff is unavailable:

1. Go to https://admin.microsoft.com and sign up for a new Microsoft 365 Business Basic subscription
2. Use a temporary domain (e.g., lasprezzatura.onmicrosoft.com) during setup
3. Create 3 user accounts: liz@, paul@, office@ (using the onmicrosoft.com domain temporarily)
4. Export email from GoDaddy M365:
   - In Outlook Web (GoDaddy login), go to Settings > General > Privacy and data > Export mailbox
   - Download the PST file for each mailbox
   - OR use IMAP migration: In new M365 admin center > Setup > Data migration > select IMAP
5. Import into new M365:
   - Use Outlook desktop app: File > Open & Export > Import/Export > Import from PST
   - OR use the M365 admin center IMAP migration wizard

### Step 3: Add lasprezz.com as Custom Domain

1. Go to https://admin.microsoft.com > Settings > Domains > Add domain
2. Enter: lasprezz.com
3. Microsoft will provide a TXT verification record (format: MS=msXXXXXXXX)
4. Add this TXT record to the CURRENT DNS provider (Wix for lasprezz.com, or wherever nameservers point)
5. Wait for Microsoft to verify (usually 5-15 minutes)
6. Set lasprezz.com as the primary domain

### Step 4: Create Mailboxes and Aliases

1. In M365 admin center > Users > Active users
2. Create/update 3 mailboxes:
   - liz@lasprezz.com (primary for Liz)
   - paul@lasprezz.com
   - office@lasprezz.com
3. Add email aliases (Users > select user > Manage email aliases):
   - liz@lasprezz.com aliases: design@lasprezz.com, team@lasprezzaturany.com, liz@lasprezzny.com
   - office@lasprezz.com aliases: hi@casadeolivier.com, office@lasprezzny.com
4. Note: aliases from other domains (lasprezzaturany.com, lasprezzny.com, casadeolivier.com) can only be added AFTER those domains are also added to the M365 tenant -- defer until DNS cutover

### Step 5: Configure DKIM

1. In M365 admin center or Microsoft Defender portal > Email authentication > DKIM
2. Select lasprezz.com
3. Note the two CNAME records Microsoft provides:
   - selector1._domainkey.lasprezz.com -> selector1-lasprezz-com._domainkey.{tenant}.onmicrosoft.com
   - selector2._domainkey.lasprezz.com -> selector2-lasprezz-com._domainkey.{tenant}.onmicrosoft.com
4. Save these values -- they will be added to Cloudflare DNS in Plan 12-02

### Step 6: Reconfigure Liz's Devices

1. Outlook mobile app (iPhone):
   - Settings > Add Account > enter liz@lasprezz.com
   - Sign in with the new Microsoft direct credentials
   - Remove the old GoDaddy-managed account after confirming the new one works
2. Outlook Web:
   - Go to https://outlook.office.com
   - Sign in with liz@lasprezz.com and new credentials
   - Verify email history is present (if tenant handoff) or import PST (if new tenant)

### Step 7: Verify

- [ ] DKIM selector CNAME values are recorded for DNS setup
- [ ] Liz can send and receive email from liz@lasprezz.com via Outlook
- [ ] Email history is accessible (either via handoff or import)
