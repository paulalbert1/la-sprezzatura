### Part A: Secondary Domain DNS Setup (repeat for each: lasprezzaturany.com, lasprezzny.com, casadeolivier.com)

For EACH secondary domain in Cloudflare:

1. **Remove all existing DNS records** except NS records (clean slate for redirect-only domains)

2. **Add proxied A records** (required for Cloudflare Redirect Rules to work):

| Type | Name | Content | Proxy | TTL |
|------|------|---------|-------|-----|
| A | @ | 192.0.2.1 | Proxied (orange cloud) | Auto |
| A | www | 192.0.2.1 | Proxied (orange cloud) | Auto |

Note: 192.0.2.1 is a dummy IP from the RFC 5737 documentation range. Traffic never reaches it -- Cloudflare intercepts it for the redirect rule.

3. **Set SSL/TLS mode** for each secondary domain:
   - Cloudflare > {domain} > SSL/TLS > Overview
   - Set to: **Full** (not Full Strict -- there is no origin server for these domains)
   - Enable "Always Use HTTPS"

### Part B: Create Cloudflare Redirect Rules

In Cloudflare, Redirect Rules are configured per-zone (per-domain). For each secondary domain:

1. Go to Cloudflare Dashboard > {domain} > Rules > Redirect Rules
2. Create a new rule:

**Rule for lasprezzaturany.com:**
- Rule name: "Redirect to lasprezz.com"
- When: Hostname equals "lasprezzaturany.com" OR Hostname equals "www.lasprezzaturany.com"
- Then: URL redirect
  - Type: Dynamic
  - Expression: `concat("https://lasprezz.com", "")`
  - Status code: 301 (Permanent)
  - Preserve query string: OFF

**Rule for lasprezzny.com:**
- Rule name: "Redirect to lasprezz.com"
- When: Hostname equals "lasprezzny.com" OR Hostname equals "www.lasprezzny.com"
- Then: URL redirect
  - Type: Dynamic
  - Expression: `concat("https://lasprezz.com", "")`
  - Status code: 301 (Permanent)
  - Preserve query string: OFF

**Rule for casadeolivier.com:**
- Rule name: "Redirect to lasprezz.com"
- When: Hostname equals "casadeolivier.com" OR Hostname equals "www.casadeolivier.com"
- Then: URL redirect
  - Type: Dynamic
  - Expression: `concat("https://lasprezz.com", "")`
  - Status code: 301 (Permanent)
  - Preserve query string: OFF

### Part C: Email Alias Forwarding

1. Go to M365 admin center > Settings > Domains
2. Add each secondary domain: lasprezzaturany.com, lasprezzny.com, casadeolivier.com
3. For each, add the Microsoft-provided TXT verification record to Cloudflare DNS for that domain
4. After verification, configure email aliases:

   - office@lasprezz.com: add aliases hi@casadeolivier.com, office@lasprezzny.com
5. Add MX records for each secondary domain pointing to M365 (so forwarding works):

| Domain | Type | Name | Content | Priority |
|--------|------|------|---------|----------|
| lasprezzaturany.com | MX | @ | {tenant}.mail.protection.outlook.com | 0 |
| lasprezzny.com | MX | @ | {tenant}.mail.protection.outlook.com | 0 |
| casadeolivier.com | MX | @ | {tenant}.mail.protection.outlook.com | 0 |

Note: MX records must NOT be proxied (Cloudflare cannot proxy MX).
