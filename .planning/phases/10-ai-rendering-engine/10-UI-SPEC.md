# Phase 10 -- UI Design Contract

> Visual and interaction contract for the AI Rendering Engine backend phase. Phase 10 is backend-first: 3 Sanity schemas, 6 API routes, a structured luxury prompt builder, Gemini API integration, and Vercel Blob storage. No new Astro pages or portal components are built in this phase. The UI contract covers Sanity Studio schema presentation, API response copywriting (consumed by Phase 11 Studio tool), siteSettings extensions, and error/status messaging. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Sanity Studio Schema Presentation

Phase 10 introduces 3 new document types and extends `siteSettings`. The schema fields are presented in Sanity Studio using `@sanity/ui` primitives. This section defines the field organization, titles, descriptions, and validation messages that Liz and Paul see in Studio.

### renderingSession Document Type

**Studio list presentation:**
- Title: "Rendering Sessions"
- List item title: `{sessionTitle}` (freeform title field, required)
- List item subtitle: `{project.title}` or "Scratchpad" if no project linked
- Hidden from default document list (managed by Phase 11 custom tool)

**Field groups:**

| Group | Label | Fields |
|-------|-------|--------|
| setup | Setup | sessionTitle, project, aspectRatio, stylePreset, description |
| inputs | Inputs | inputs.images[] |
| renderings | Renderings | renderings[], conversation[] |
| metadata | Metadata | status, createdBy, createdAt, lastError |

**Key field specifications:**

| Field | Type | Title | Description | Validation |
|-------|------|-------|-------------|------------|
| sessionTitle | string | Session Title | A descriptive name for this rendering session | Required. "Give this session a descriptive name." |
| project | reference (project) | Project | Link to a client project, or leave empty for scratchpad | Optional. |
| aspectRatio | string (list) | Aspect Ratio | Output image proportions | Required. Options: "16:9" (default), "1:1", "4:3". Initial value: "16:9". |
| stylePreset | string | Design Style | Style direction for the rendering (e.g., "quiet luxury", "modern European", "transitional") | Optional. Description: "Enter a design style or leave empty for the AI to interpret from your description." |
| description | text | Design Vision | Describe what you want the rendering to show | Required. Description: "Write naturally -- the system structures your description for the AI." Rows: 4. |
| status | string (list) | Status | Current session status | Read-only in Studio. Options: "idle", "generating", "complete", "error". Initial value: "idle". |
| lastError | string | Last Error | Most recent error message | Read-only in Studio. Hidden when empty. |
| createdBy | string | Created By | Sanity user ID of the session creator | Read-only. Hidden from Studio form (set by API). |
| createdAt | datetime | Created At | Session creation timestamp | Read-only. Hidden from Studio form (set by API). |

**inputs.images[] array member:**

| Field | Type | Title | Description | Validation |
|-------|------|-------|-------------|------------|
| blobPathname | string | Image Path | Vercel Blob pathname | Required. Read-only in Studio (set by BlobFileInput). |
| imageType | string (list) | Image Type | What this image represents | Required. Options loaded from `siteSettings.renderingImageTypes`. Default types: "Floor Plan", "Existing Space Photo", "Inspiration", "Material/Finish Sample", "Furniture Reference", "Fixture Reference", "Custom". |
| location | string | Placement | Where this element goes in the room (e.g., "left wall near window") | Optional. Description: "Describe the relative position in the room." |
| notes | string | Notes | Additional context for the AI about this image | Optional. |
| copyExact | boolean | Replicate Exactly | When checked, the AI will copy this item exactly rather than interpreting the style | Initial value: inferred from imageType (true for Furniture/Fixture/Material references, false for Inspiration/Floor Plan/Existing Space/Custom). |

**renderings[] array member (renderingOutput):**

| Field | Type | Title | Description |
|-------|------|-------|-------------|
| blobPathname | string | Image Path | Vercel Blob pathname of the generated image |
| prompt | text | Composed Prompt | The full prompt sent to Gemini (hidden from Studio form, stored for debugging) |
| textResponse | text | AI Response | Text returned by Gemini alongside the image |
| isPromoted | boolean | Promoted | Whether this rendering has been promoted to a Design Option |
| generatedAt | datetime | Generated At | When this rendering was created |
| status | string (list) | Status | "complete", "error", "generating" |
| errorMessage | string | Error | Error details if generation failed |
| modelId | string | Model | Gemini model used |
| latencyMs | number | Latency (ms) | Generation time in milliseconds |
| inputTokens | number | Input Tokens | Tokens consumed by input |
| outputTokens | number | Output Tokens | Tokens in the response |
| costEstimate | number | Cost (cents) | Estimated cost in integer cents |
| bytesStored | number | Image Size (bytes) | Size of the stored image |

**conversation[] array member:**

| Field | Type | Title | Description |
|-------|------|-------|-------------|
| role | string (list) | Role | "user" or "model" |
| text | text | Message | The conversation message text |
| image | string | Image | Optional Blob pathname of an attached image |
| timestamp | datetime | Time | When this message was sent |

## Copywriting Contract

### API Response Messages (consumed by Phase 11 Studio tool)

| Element | Copy |
|---------|------|
| Primary CTA | Generate Rendering |
| Generation started (202 response) | Generating your rendering... This may take 15-30 seconds. |
| Generation complete | Rendering complete |
| Refinement started (202 response) | Refining your rendering... |
| Usage check response | {count} of {limit} renderings used this month. {remaining} remaining. |

### Error Messages (API responses)

| Element | Copy |
|---------|------|
| Unauthorized (401) | Invalid studio token. Please reload Sanity Studio and try again. |
| User excluded (403) | Your account does not have access to AI Rendering. |
| Quota exceeded (403) | Monthly rendering limit reached ({count}/{limit}). Contact Paul to adjust your allocation. |
| API not configured (503) | AI Rendering is not configured. Contact Paul to set up the Gemini API key. |
| Content policy rejection | The AI flagged a potential issue with your content or imagery. Try rephrasing your description or using different reference images. |
| API timeout/5xx | Generation failed due to a server error. Tap to retry -- you will not be charged for the failed attempt. |
| Blob upload failure (after successful generation) | The rendering was generated but could not be saved. Retrying upload... |
| No image in response | The AI returned text but no image. This can happen with certain prompts. Try rephrasing your description. |
| Invalid session (404) | Rendering session not found. It may have been deleted. |
| Invalid input images | One or more input images could not be found. Please re-upload and try again. |
| Missing description (400) | Please describe your design vision before generating. |
| Missing session title (400) | Please give your session a descriptive name. |

### Empty State Messages

| Element | Copy |
|---------|------|
| No rendering sessions (Phase 11 tool) | No rendering sessions yet. Create your first session to start generating AI room visualizations. |
| No renderings in session | No renderings generated yet. Describe your design vision and generate your first rendering. |
| No design options for project | No design options have been shared for this project yet. |
| Empty usage (no document) | No renderings generated this month. {limit} available. |

### Validation Messages (Sanity Schema)

| Field | Validation | Message |
|-------|-----------|---------|
| sessionTitle | Required | A session title is required |
| description | Required | A design vision description is required |
| aspectRatio | Required | Select an aspect ratio for this session |
| inputs.images[].imageType | Required | Select what this image represents |
| inputs.images[].blobPathname | Required | An image file is required |
| designOption.project | Required | A project must be selected |
| designOption.blobPathname | Required | An image is required |
| renderingUsage.count | Min 0, integer | Count must be a non-negative integer |
| renderingUsage.limit | Min 1, integer | Limit must be a positive integer |
| renderingUsage.bytesStored | Min 0, integer | Bytes stored must be a non-negative integer |
| costEstimate | Min 0, integer | Cost must be a non-negative integer (cents) |

### Soft Warning Messages

| Trigger | Copy |
|---------|------|
| No floor plan in session inputs | Results may be less spatially accurate without a floor plan. You can still generate -- some sessions are styling-focused. |
| Approaching quota (>=80% used) | You have used {count} of {limit} renderings this month. {remaining} remaining. |

### Destructive Action Confirmations

| Action | Confirmation Copy |
|--------|------------------|
| Unpromote Design Option | Remove this design option? The client will no longer see it. The original rendering is preserved in the session. |
| Delete rendering session | This action is not available in Phase 10. Sessions are never deleted (per CONTEXT.md "keep everything" decision). |

## API Response Shape Contract

Phase 11 consumes these JSON response shapes. Defined here to ensure consistency between backend (Phase 10) and frontend (Phase 11).

### POST /api/rendering/generate

**Request:**
```json
{
  "sessionId": "string (existing session ID, or omit to create new)",
  "sessionTitle": "string (required if creating new)",
  "projectId": "string | null",
  "aspectRatio": "16:9 | 1:1 | 4:3",
  "stylePreset": "string | null",
  "description": "string (required)",
  "images": [
    {
      "blobPathname": "string",
      "imageType": "string",
      "location": "string | null",
      "notes": "string | null",
      "copyExact": "boolean"
    }
  ],
  "sanityUserId": "string"
}
```

**Response (202 Accepted):**
```json
{
  "status": "generating",
  "sessionId": "string",
  "message": "Generating your rendering... This may take 15-30 seconds."
}
```

**Error (403 Quota):**
```json
{
  "error": "Monthly rendering limit reached (50/50). Contact Paul to adjust your allocation.",
  "code": "QUOTA_EXCEEDED",
  "usage": { "count": 50, "limit": 50, "remaining": 0 }
}
```

### POST /api/rendering/refine

**Request:**
```json
{
  "sessionId": "string (required)",
  "refinementText": "string (required)",
  "newImages": [
    {
      "blobPathname": "string",
      "imageType": "string",
      "location": "string | null",
      "notes": "string | null",
      "copyExact": "boolean"
    }
  ],
  "sanityUserId": "string"
}
```

**Response (202 Accepted):**
```json
{
  "status": "generating",
  "sessionId": "string",
  "message": "Refining your rendering..."
}
```

### GET /api/rendering/status?sessionId={id}

**Response (200):**
```json
{
  "status": "idle | generating | complete | error",
  "sessionId": "string",
  "latestRendering": {
    "blobPathname": "string | null",
    "textResponse": "string | null",
    "status": "complete | error | generating",
    "errorMessage": "string | null",
    "generatedAt": "ISO datetime | null"
  },
  "renderingCount": "number"
}
```

### GET /api/rendering/usage?sanityUserId={id}

**Response (200):**
```json
{
  "sanityUserId": "string",
  "month": "2026-03",
  "count": "number",
  "limit": "number",
  "remaining": "number",
  "bytesStored": "number"
}
```

### POST /api/rendering/promote

**Request:**
```json
{
  "sessionId": "string",
  "renderingIndex": "number",
  "projectId": "string (required -- if scratchpad, must be provided)",
  "caption": "string | null",
  "sanityUserId": "string"
}
```

**Response (200):**
```json
{
  "success": true,

  "message": "Rendering promoted to Design Options."

```

### POST /api/rendering/promote (unpromote)

**Request:**
```json

  "sessionId": "string",

```

## Interaction Contract

### Generation Flow (API level)

```
[Phase 11 Studio tool sends POST /api/rendering/generate]
  --> [API: validate x-studio-token header]
    --> 401 if invalid
  --> [API: check exclude list via siteSettings.renderingExcludedUsers]
    --> 403 if excluded
  --> [API: check GEMINI_API_KEY exists]
    --> 503 if missing
  --> [API: check usage quota]
    --> 403 with usage object if exceeded
  --> [API: validate input images (Blob pathnames exist)]
    --> 400 if invalid
  --> [API: create/update renderingSession with status: "generating"]
  --> [API: return 202 immediately]
  --> [waitUntil: fetch input images from Blob, resize for Gemini]
  --> [waitUntil: build luxury prompt via promptBuilder]
  --> [waitUntil: call Gemini generateContent]
    --> [Success: extract image, upload to Blob, append to renderings[], add conversation entry, increment usage, set status: "complete"]
    --> [Content policy: set status: "error", store error message, do NOT increment usage]
    --> [No image in response: set status: "error", store error message, do NOT increment usage]
    --> [Timeout/5xx: set status: "error", store error message, do NOT increment usage]
    --> [Blob upload failure after image generated: store base64 temporarily, retry upload, set status: "error" if retry fails]
```

### Refinement Flow (API level)

```
[Phase 11 Studio tool sends POST /api/rendering/refine]
  --> [API: validate auth, check quota (same as generate)]
  --> [API: fetch existing session with full conversation history]
  --> [API: set status: "generating", return 202]
  --> [waitUntil: reconstruct conversation contents array from stored history]
  --> [waitUntil: fetch previous output image from Blob for context]
  --> [waitUntil: append refinement text with integration instructions]
  --> [waitUntil: call Gemini generateContent with full contents]
    --> [Same success/error handling as generate flow]
```

### Polling Flow

```
[Phase 11 Studio tool polls GET /api/rendering/status?sessionId={id}]
  --> [API: validate auth]
  --> [API: fetch session using sanityWriteClient (bypasses CDN cache)]
  --> [Return current status, latest rendering info, total count]
  --> [Phase 11 client: poll every 3 seconds while status === "generating"]
  --> [Phase 11 client: stop polling when status === "complete" or "error"]
```

### Promote Flow

```
[Phase 11 Studio tool sends POST /api/rendering/promote]
  --> [API: validate auth]
  --> [API: fetch session, verify renderingIndex exists]
  --> [If scratchpad session and no projectId provided: return 400 "Select a project to promote this rendering"]
  --> [API: create designOption document with provenance]
  --> [API: set renderings[index].isPromoted = true on session]
  --> [Return designOptionId]
```

### Client Reaction Flow

```
[Portal client sends POST /api/rendering/react]
  --> [API: validate portal session auth (cookie, not Studio token)]
  --> [API: fetch designOption, verify client has access to project]
  --> [Favorite: toggle reaction entry on designOption.reactions[]]
  --> [Comment: append to reactions[]]
  --> [Return success]
```
