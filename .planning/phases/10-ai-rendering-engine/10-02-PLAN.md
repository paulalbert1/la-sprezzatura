3. `buildImageRoleLabel(imageType: string, input: Partial<ImageInput>): string` function:
Returns the role label text for an image part in the Gemini prompt. Format:
- `"FLOOR PLAN: Use this as the spatial layout reference."` for Floor Plan
- `"EXISTING SPACE PHOTO: This is the current state of the room. Preserve architecture and lighting direction."` for Existing Space Photo
- `"INSPIRATION: Apply the style and mood, not exact replication."` for Inspiration (when copyExact is false)
- `"FURNITURE REFERENCE [left wall near window]: Replicate this piece exactly."` for Furniture Reference with location and copyExact true
- `"MATERIAL/FINISH SAMPLE: Replicate this material exactly."` for Material/Finish Sample (copyExact true)
- `"FIXTURE REFERENCE: Replicate this fixture exactly."` for Fixture Reference (copyExact true)
- Generic format for Custom and unknown types: `"{TYPE}: {notes or generic instruction}"`
Add location in brackets when present. Add notes appended after the main instruction when present.

4. `buildLuxuryPrompt(description: string, images: ImageInput[], stylePreset?: string): string` function:
Assembles the 12-section luxury template. Hardcoded sections (per CONTEXT.md decision):

Section 1 - Preserve: `"## Preserve\nLock the architecture, layout, camera angle, and lighting direction from the base room image. Do not alter walls, ceiling, flooring, or window positions unless explicitly instructed."`

Section 2 - Design Intent: `"## Design Intent\nStyle: ${stylePreset || "quietly luxurious"}. Create an interior that embodies this aesthetic with restraint and sophistication."`

Section 3 - Primary Change: `"## Primary Change\n${description}"`

Section 4 - Reference Binding: List each image with its copy/interpret instruction derived from the image metadata. Format: `"- ${imageType.toUpperCase()}: ${copyExact ? "Replicate exactly" : "Apply style/concept, not exact copy"}${notes ? ". " + notes : ""}"`. If no images, section reads "No reference images provided -- interpret the description freely."

Section 5 - Placement & Composition: Only included when at least one image has a `location` value. Lists: `"- ${imageType}: ${location}"` for each image with location.

Section 6 - Scale & Proportion: `"## Scale & Proportion\nMaintain realistic scale relationships. All furniture and objects must be proportional to room dimensions. Human-scale reference points preserved."`

Section 7 - Material & Detail Fidelity: `"## Material & Detail Fidelity\nRender all materials with photographic realism. No plastic, glossy, or synthetic appearance. Show natural grain, texture, patina, and wear appropriate to the material."`

Section 8 - Lighting & Atmosphere: `"## Lighting & Atmosphere\nSoft, natural, diffused lighting. Realistic shadow falloff with warm ambient tones. Light sources should be consistent with window positions and time of day implied by the base room."`

Section 9 - Styling Restraint: `"## Styling Restraint\nDo NOT over-decorate. Leave breathing room. Negative space is intentional. Every object must serve a purpose. Remove anything that feels cluttered or gratuitous."`

Section 10 - Color & Cohesion: `"## Color & Cohesion\nHarmonious palette. New elements must integrate seamlessly with existing space colors. No jarring contrasts unless explicitly requested."`

Section 11 - Constraints: `"## Constraints\nDo not modify locked architectural elements. Do not introduce clutter, excessive accessories, or items not mentioned in the description. Do not add people, pets, or text."`

Section 12 - Final Quality: `"## Final Quality\nEditorial-quality, cohesive, realistic, quietly luxurious. The result should be suitable for Architectural Digest or a luxury design portfolio."`

Join all sections with `"\n\n"`.

3. `fetchAndEncodeImage(blobPathname: string): Promise<{ base64: string; mimeType: string }>` function:
- Uses `get` from `@vercel/blob` to fetch the image by pathname
- Converts the response body to an ArrayBuffer via `.arrayBuffer()`

- If `get` returns null/undefined, throw `Error("Image not found: ${blobPathname}")`

```typescript
  prompt: string;
```

Implementation:

Exports:

```typescript
export interface AuthResult {
```

```typescript
export interface UsageResult {
  allowed: boolean;
  count: number;
  limit: number;
  remaining: number;
}
```

3. `validateRenderingAuth(request: Request): Promise<AuthResult>` function:
- Check `x-studio-token` header against `import.meta.env.STUDIO_API_SECRET`
- If missing/mismatch: return `{ authorized: false, error: "Invalid studio token. Please reload Sanity Studio and try again.", statusCode: 401 }`
- Clone request and parse JSON body to extract `sanityUserId`
- If no sanityUserId: return `{ authorized: false, error: "Missing sanityUserId in request body.", statusCode: 400 }`
- Fetch `renderingExcludedUsers` from siteSettings via `sanityWriteClient.fetch(RENDERING_SETTINGS_QUERY)`
- If user in excluded list: return `{ authorized: false, error: "Your account does not have access to AI Rendering.", statusCode: 403 }`
- Check `import.meta.env.GEMINI_API_KEY` -- if not set AND `import.meta.env.RENDERING_TEST_MODE` is not "true": return `{ authorized: false, error: "AI Rendering is not configured. Contact Paul to set up the Gemini API key.", statusCode: 503 }`
- Return `{ authorized: true, sanityUserId }`

4. `checkUsageQuota(sanityUserId: string): Promise<UsageResult>` function:
- Compute month: `new Date().toISOString().slice(0, 7)` (e.g., "2026-03")
- Compute docId: `usage-${sanityUserId}-${month}`
- Fetch usage doc via sanityWriteClient. If not found:
  - Fetch `renderingAllocation` from siteSettings (default 50)
  - Create new renderingUsage document with `_id: docId`, count: 0, limit, bytesStored: 0
- Compute remaining = limit - count
- Return `{ allowed: remaining > 0, count, limit, remaining: Math.max(0, remaining) }`

5. `incrementUsage(sanityUserId: string, bytesStored: number): Promise<void>` function:
- Compute docId same way
- Call `sanityWriteClient.patch(docId).inc({ count: 1, bytesStored }).commit()`
