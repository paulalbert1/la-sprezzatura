# Phase 20 — UI Design Contract

> Visual and interaction contract for the Portfolio Project Type phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Sanity Studio phase, not public site) |
| Preset | not applicable |
| Component library | @sanity/ui 3.1.13 (Dialog, Grid, Card, Checkbox, Button, Stack, Flex, Text, Box, Tooltip) |
| Icon library | @sanity/icons 3.7.4 (StarFilledIcon, ImageIcon, CheckmarkIcon, DocumentsIcon) |
| Font | Sanity Studio default (Inter via @sanity/ui); public site uses DM Sans / Cormorant Garamond but is not in scope |

**Note:** This phase operates entirely within Sanity Studio. All UI follows existing Studio conventions, the custom warm stone theme (`studioTheme.ts`), and CSS overrides (`studio.css`). No shadcn, no Tailwind, no public-facing pages.

---

## Spacing Scale

Declared values (must be multiples of 4). All spacing uses `@sanity/ui` `padding`, `gap`, and `margin` props with Sanity's integer scale (1=4px, 2=8px, 3=12px, 4=16px, 5=20px).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (space=1) | Checkbox-to-label gap, inline icon padding |
| sm | 8px (space=2) | Compact element spacing, button gaps |
| md | 16px (space=4) | Dialog content padding, section spacing within modal |
| lg | 24px (space=6) | Gap between modal sections (photography question, image grid, buttons) |
| xl | 32px (space=8) | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Image thumbnail grid uses `gap={3}` (12px) for tighter visual density in the selection grid. This matches the existing `StepUpload.tsx` pattern.

---

## Typography

All typography within the spawn modal uses `@sanity/ui` `Text` and `Heading` components with their `size` prop. Inline styles follow the established `sendUpdate.tsx` pattern for dialog content.

| Role | Size | Weight | Line Height | Implementation |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `<Text size={2}>` or `fontSize: "14px"` inline |
| Label | 13px | 500 (medium) | 1.4 | `<Text size={1}>` or `fontSize: "13px"` with `fontWeight: 500` inline |
| Heading | 16px | 600 (semibold) | 1.2 | Dialog `header` prop (rendered by Sanity's Dialog chrome) |
| Caption | 12px | 500 (medium) | 1.3 | `<Text size={0}>` — image labels ("Hero", "Photo 2"), helper text |

## Color

All colors derive from the existing warm stone palette in `studioTheme.ts` and `studio.css`. No new colors are introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FAF5F2 (warm off-white) | Dialog background (inherits from Studio theme), modal surface |
| Secondary (30%) | #F0EEEB (warm gray) | Selected image card background, preview section background, photography question card |
| Accent (10%) | #D97706 (amber/gold) | Hero star badge, "Create" primary CTA button background, selected image card border |
| Destructive | #EF4444 (red-500) | Error state banner only (creation failure) |

Accent reserved for:
- Hero image star icon (#D97706 amber)
- "Create Portfolio Version" primary CTA button (uses `tone="positive"` from Sanity, which renders green in default; override to amber via inline style matching `#92400E` CTA pattern from studio.css)
- Selected image card tone (`tone="primary"` on Card, which maps to amber via studio.css overrides)

Additional semantic colors:
- Warning: #FEF3C7 background / #92400E text (amber warning banner) — used if photography question needs clarification
- Success: #22C55E (green-500) — toast notification after successful portfolio creation
- Muted text: #78716C (stone-500) — helper text, captions, secondary labels

## Interaction States

### Image Selection

| State | Visual Treatment |
|-------|-----------------|
| Default (unselected) | `Card tone="default"`, no border highlight, unchecked checkbox |
| Selected | `Card tone="primary"` (amber border via studio.css), checked checkbox |
| Hero image | Star badge overlay (always visible), pre-checked by default, cannot be unchecked while other images are selected |
| Hover | Cursor pointer on thumbnail, Card hover state from @sanity/ui |

### Dialog Flow

| State | What Happens |
|-------|--------------|
| Dialog opens | Step 1 (photography question) shown. Dialog header: "Create Portfolio Version" |
| User clicks "Yes, select photos" | Transition to Step 2 (image grid). All images pre-loaded via `imageUrlBuilder`. Hero pre-checked with star badge. |
| User clicks "No, create without photos" | Skip to creation. Portfolio created with text fields only. No images. |
| User clicks "Create Portfolio Version" | Transition to Step 3 (creating). Spinner + status text. Button disabled. |
| Creation succeeds | Dialog closes. Success toast: "Portfolio version created". Navigate to new document via `window.location.href = /admin/structure/portfolioProject;{newDocId}` |
| Creation fails | Error banner shown in dialog (red). Button re-enabled. User can retry. |
| User clicks Cancel (any step) | Dialog closes. No changes made. |

### Document Action Visibility

| Condition | Action Visible? |
|-----------|----------------|
| `projectStatus !== "completed"` | No — action returns `null` |
| `projectStatus === "completed"` AND no existing portfolioProject with matching `sourceAdminProjectId` | Yes — button shown |
| `projectStatus === "completed"` AND portfolioProject already exists | No — duplicate check returns `null` |
| Duplicate check loading | No — hide until check resolves (prevents flash of button) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Dialog traps focus (built into @sanity/ui Dialog). Tab through buttons and checkboxes. Enter/Space to toggle. |
| Image selection | Each image card is clickable (onClick on img and Card). Checkbox is a native `<input type="checkbox">` for assistive tech. |
| Star badge | Decorative only (hero is identified by label text "Hero"). No ARIA role needed on the icon. |
| Button labels | All buttons have explicit `text` prop. No icon-only buttons without accessible labels. |
| Error announcement | Error banner uses `role="alert"` for screen reader announcement. |
| Color contrast | All text meets WCAG AA on warm off-white (#FAF5F2). Body text #0F172A on #FAF5F2 = 15.2:1. Muted text #78716C on #FAF5F2 = 4.6:1. Amber accent #D97706 on #FAF5F2 = 4.5:1 (passes AA for normal text). |
