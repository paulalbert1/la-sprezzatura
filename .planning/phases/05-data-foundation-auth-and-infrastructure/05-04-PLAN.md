    This is a human-action checkpoint. Claude cannot perform these steps -- they require dashboard access to Vercel, Resend, and the DNS provider.

    **Upstash Redis Setup (required for auth to work):**
    1. Go to Vercel Dashboard -> your la-sprezzatura project -> Storage tab
    2. Click "Create" -> select "Upstash Redis"
    3. Choose the free tier, name it "la-sprezzatura-redis"
    4. Vercel auto-populates UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN as env vars
    5. Verify the env vars are visible in Vercel Dashboard -> Settings -> Environment Variables
    6. Also add these same values to your local .env file for local dev:
       - UPSTASH_REDIS_REST_URL=https://...
       - UPSTASH_REDIS_REST_TOKEN=AX...

    **Resend Domain Verification (required for production email delivery):**
    1. Go to Resend Dashboard (resend.com/domains)
    2. Click "Add Domain"
    3. Enter: send.lasprezz.com
    4. Resend will generate DNS records -- typically:
       - 1 MX record for send.lasprezz.com
       - 1 TXT record (SPF) for send.lasprezz.com
       - 3 CNAME records (DKIM) for various selectors
    5. Copy each record and add them to the DNS provider for lasprezz.com
    6. Wait for DNS propagation (can take up to 48 hours, usually 15-60 minutes)
    7. Return to Resend Dashboard and click "Verify" -- status should change to "Verified"

    NOTE: DNS propagation may take time. You can proceed to Task 2 while waiting -- the auth flow works with the sandbox sender in the meantime.
