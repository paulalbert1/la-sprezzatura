## Component Inventory

### Page 1: Login (`/portal/login`)

**Layout:** Centered single-column form, max-width 400px, vertically centered in viewport. Wrapped in `PortalLayout.astro`.

**Elements:**
- Brand mark: `text-xs text-stone tracking-[0.2em] uppercase` -- "La Sprezzatura" (matches existing portal branding pattern)
- Heading: `font-heading text-4xl md:text-5xl font-light text-charcoal text-center` -- "Welcome to Your Portal"
- Subtitle: `text-base text-stone font-body text-center` -- "Enter your email to access your projects"
- Email input: follows `ContactForm.tsx` Field pattern -- `bg-cream-dark border border-stone-light px-4 py-3 font-body text-sm` with `focus:ring-2 focus:ring-terracotta`
- Label: `text-xs uppercase tracking-widest text-stone mb-2 font-body` -- "Email Address"
- Submit button: `bg-terracotta text-white text-xs uppercase tracking-widest font-body px-10 py-4 w-full hover:bg-terracotta-light transition-colors disabled:opacity-60 disabled:cursor-not-allowed` -- "Continue"
- Loading state: button text changes to "Sending..." with `disabled` attribute
- Help text below form: `text-sm text-stone text-center mt-6` -- "We'll send a secure access link to your email"

**States:**
- Idle: form ready with empty email field
- Submitting: button shows "Sending...", disabled
- Success: form replaced by confirmation message (see Copywriting Contract)
- Error (validation): red border on email field, inline error text
- Error (rate limit): error banner above form (matches existing `ContactForm.tsx` error banner pattern: `border border-terracotta/30 bg-terracotta/5 px-5 py-4`)
- Error (server): same error banner pattern with generic message

**Interaction:** Form submission via Astro Action (React component with `client:load`). On success, replace entire form area with confirmation state -- no page navigation. On error, stay on form with error displayed.

### Page 2: Verify (`/portal/verify`)

**Layout:** Minimal -- this is a transient redirect page. Users see it briefly while token is consumed.

**Elements:**
- Brand mark: same as login
- Heading: "Verifying..." (visible only during the brief server-side processing)
- If token valid: immediate redirect to `/portal/dashboard` (client never sees content)
- If token expired/invalid: redirect to `/portal/login?error=expired`

**States:**
- Verifying: brief loading indicator (not a full page -- SSR handles this server-side before HTML is sent)
- Expired: redirects to login with query parameter triggering error banner
- Invalid: same as expired

### Page 3: Dashboard (`/portal/dashboard`)

**Layout:** Centered single-column, max-width 640px (matches existing `max-w-2xl` portal pattern). Wrapped in `PortalLayout.astro`.

**Elements:**
- Brand mark: same as login/existing portal
- Greeting: `font-heading text-4xl md:text-5xl font-light text-charcoal text-center` -- "Welcome back, {firstName}"
- Subtitle: `text-lg text-stone font-body text-center mb-8` -- "{N} active project(s)"
- Project cards: grid of clickable cards, single column on mobile, 2 columns on desktop for 2+ projects

**Project Card Design:**
- Container: `bg-cream-dark border border-stone-light/20 p-6 hover:border-terracotta/40 transition-colors cursor-pointer`
- No box-shadow (matches luxury minimal aesthetic -- shadows are not used in the existing portal/public site)
- Project name: `font-heading text-xl font-light text-charcoal`
- Pipeline stage badge: reuse existing `StatusBadge.astro` component
- Engagement type: `text-xs text-stone uppercase tracking-wider font-body` -- displays "Full Interior Design", "Styling & Refreshing", or "Carpet Curating"
- Completed project treatment: card has `opacity-60` on entire card, badge shows completed state (`bg-emerald-50 text-emerald-700`)

**Card ordering:** Active projects first, completed projects below with a `text-xs text-stone uppercase tracking-widest` divider label "Completed Projects" between groups (only shown if completed projects exist).

**Auto-redirect:** If client has exactly 1 project, server-side redirect to project detail view (`/portal/project/{id}`) -- dashboard page never renders.

**Logout:** Text link in footer area -- `text-xs text-stone hover:text-terracotta transition-colors uppercase tracking-widest` -- "Sign Out". Clears session cookie and redirects to login.

### Page 4: PURL Redirect (`/portal/[token]`)

**Layout:** Centered message, same pattern as existing 404 state in `[token].astro`.

**Elements:**
- Heading: `font-heading text-3xl font-light text-charcoal mb-4` -- "We've Upgraded Your Portal"
- Body: `text-stone font-body leading-relaxed` -- "For your security, we've added email verification. Enter your email address to access your projects."
- CTA link: styled as outline button (`border border-stone-light/40 px-6 py-2.5 text-sm tracking-widest uppercase text-stone hover:text-terracotta hover:border-terracotta transition-colors`) -- "Go to Login" linking to `/portal/login`

### Page 5: Magic Link Email Template

**Layout:** HTML email with inline CSS, max-width 600px, centered. Follows existing email template pattern in `src/actions/index.ts`.

**Elements:**
- Background: `#FAF8F5` (cream) body, `#FFFFFF` (white) content card
- Brand: "La Sprezzatura" in uppercase tracking, Cormorant Garamond (web-safe fallback: Georgia, serif)
- Heading: "Your Portal Access Link" -- 24px, Georgia (email-safe fallback), `#2C2926`
- Body text: "Click the button below to access your project portal. This link expires in 15 minutes and can only be used once." -- 16px, system-ui (email-safe fallback), `#2C2926`
- CTA button: `background-color: #C4836A; color: #FFFFFF; padding: 16px 32px; text-decoration: none; font-size: 14px; letter-spacing: 0.1em; text-transform: uppercase;` -- "Access Your Portal"
- Fallback link: plain text URL below button -- "If the button doesn't work, copy and paste this link:" followed by full URL in `#8A8478`
- Footer: "This is an automated message from La Sprezzatura. If you didn't request this link, you can safely ignore this email." -- 12px, `#B8B0A4`
- Sender: "La Sprezzatura <noreply@send.lasprezz.com>"
- Subject: "Your La Sprezzatura Portal Access"

## Interaction Contract

### Login Flow States

```
[Idle] --submit--> [Submitting] --success--> [Success Confirmation]
                               \--error----> [Error Banner + Idle Form]
```

- Form is a React component (`client:load`) using Astro Actions
- On submit: disable button, show "Sending..." text
- On success: replace form with confirmation message (no redirect)
- On error: show error banner above form, re-enable button
- "Email not found" is intentionally treated as success (user enumeration prevention) -- always show "Check Your Email"
- Rate limit responses show rate limit error message in the same error banner

### Dashboard Interaction

- Project cards are `<a>` tags wrapping the card content (entire card is clickable)
- Hover: border transitions from `stone-light/20` to `terracotta/40`
- No JavaScript required for dashboard (static Astro page with SSR data)
- Auto-redirect for single-project clients happens server-side (never renders dashboard)

### Session Persistence

- 30-day httpOnly cookie -- client is automatically recognized on return visits
- Expired session: middleware redirects to `/portal/login` transparently
- Sign Out: clears cookie and redirects to `/portal/login`

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Focus visible | 2px solid terracotta outline, 3px offset (existing `global.css` rule) |
| Touch targets | Minimum 44px height on login button and project cards |
| Form errors | `role="alert"` on error messages (existing ContactForm pattern) |
| Color contrast | Charcoal on cream = 12.5:1 (AAA). Stone on cream = 3.6:1 (AA for large text only -- used for labels/helper text). Terracotta on white button = 4.5:1 (AA). |
| Screen reader | Login form uses `<label>` with `htmlFor`. Error messages use `role="alert"`. |
| Keyboard nav | Tab order: email input, submit button. Enter submits form. |
| Reduced motion | No animations on portal pages (portal uses static layouts, no GSAP) |
| noindex/nofollow | All portal pages include `<meta name="robots" content="noindex, nofollow">` via PortalLayout.astro |
