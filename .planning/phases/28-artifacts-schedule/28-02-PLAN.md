Create `src/pages/api/admin/artifact-version.ts` with `export const prerender = false` and a POST handler.

The POST handler supports two actions via FormData or JSON body:

**Action: "upload" (FormData)**
- Parse FormData: `file` (File), `projectId` (string), `artifactKey` (string), `note` (string, optional), `action` = "upload"
- Auth: `getSession(cookies)` + `session.role !== "admin"` -> 401
- Validate: file exists and is a File instance, projectId and artifactKey are non-empty strings
- File size limit: reject files > 25MB (25 * 1024 * 1024 bytes) with 400 "File too large (25MB max)"
- Upload the file to Sanity asset pipeline (per D-02 amended) because the schema `versions[].file` is type `"file"` which expects Sanity asset references. Use:
  ```typescript
  const buffer = Buffer.from(await file.arrayBuffer());
  const asset = await sanityWriteClient.assets.upload("file", buffer, {
    filename: file.name,
    contentType: file.type,
  });
  ```
- Generate `versionKey` via `generatePortalToken(8)`
- Build version object:
  ```typescript
  const version = {
    _key: versionKey,
    _type: "artifactVersion",
    file: { _type: "file", asset: { _type: "reference", _ref: asset._id } },
    uploadedAt: new Date().toISOString(),
    note: note || "",
  };
  ```
- Build decision log entry:
  ```typescript
  const logEntry = {
    _key: generatePortalToken(8),
    _type: "decisionEntry",
    action: "version-uploaded",
    timestamp: new Date().toISOString(),
  };
  ```
- Patch Sanity:
  ```typescript
  await sanityWriteClient
    .patch(projectId)
    .append(`artifacts[_key=="${artifactKey}"].versions`, [version])
    .append(`artifacts[_key=="${artifactKey}"].decisionLog`, [logEntry])
    .set({ [`artifacts[_key=="${artifactKey}"].currentVersionKey`]: versionKey })
    .commit();
  ```
- Return 200 with `{ success: true, versionKey, assetUrl: asset.url }`

**Action: "set-current" (JSON body)**
- Parse JSON body: `{ action: "set-current", projectId, artifactKey, versionKey }`
- Auth check (same pattern)
- Validate: projectId, artifactKey, versionKey are non-empty strings
- Patch Sanity:
  ```typescript
  await sanityWriteClient
    .patch(projectId)
    .set({ [`artifacts[_key=="${artifactKey}"].currentVersionKey`]: versionKey })
    .commit();
  ```
- Return 200 with `{ success: true }`

**Action: "upload-signed" (FormData) -- for contract signed files**
- Same auth and file validation as "upload"
- Upload to Sanity asset pipeline (same as version upload)
- Patch Sanity:
  ```typescript
  await sanityWriteClient
    .patch(projectId)
    .set({
      [`artifacts[_key=="${artifactKey}"].signedFile`]: {
        _type: "file",
        asset: { _type: "reference", _ref: asset._id },
      },
    })
    .commit();
  ```
- Return 200 with `{ success: true, assetUrl: asset.url }`

Error handling: wrap each action in try/catch, log errors with `console.error("Failed to [action] artifact version:", err)`, return 500 with `{ error: "Failed to [action description]" }`.

Determine action from: if request content-type includes "multipart/form-data", parse as FormData and read `action` field; otherwise parse as JSON and read `action` property.

        reservations
      }
    }
  }
`;
