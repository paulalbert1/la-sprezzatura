**POST handler** accepts JSON body:
```typescript
type ScheduleDateBody = {
  projectId: string;
  taskId: string;         // format "category:_key" e.g. "contractor:c1", "milestone:m1", "event:e1"
  startDate: string;      // "YYYY-MM-DD"
  endDate?: string | null; // "YYYY-MM-DD" or null for point events
  // For milestones, also accept:
  isComplete?: boolean;
};
```

**Implementation:**
1. Auth check: `getSession(cookies)` + `session.role !== "admin"` -> 401
2. Parse JSON body with try/catch -> 400 "Invalid JSON" on failure
3. Validate required fields: projectId, taskId, startDate are non-empty strings -> 400 "Missing required fields"
4. Parse taskId: `const [category, _key] = taskId.split(":");`
5. Validate category is one of: "contractor", "milestone", "event" -- reject "procurement" with 400 "Procurement dates are read-only" (per D-08)
6. Validate _key is non-empty
7. Validate startDate matches YYYY-MM-DD format (regex: `/^\d{4}-\d{2}-\d{2}$/`)
8. If endDate is provided, validate same format

**Field mapping** (per RESEARCH.md):
```typescript
const fieldMap: Record<string, { array: string; startField: string; endField: string }> = {

  event:      { array: "customEvents", startField: "date", endField: "endDate" },
};
```

**Build Sanity patch:**
```typescript
const mapping = fieldMap[category];
if (!mapping) {
  return new Response(JSON.stringify({ error: "Invalid category" }), { status: 400, ... });

const setObj: Record<string, unknown> = {};
setObj[`${mapping.array}[_key=="${_key}"].${mapping.startField}`] = startDate;
```

The POST handler supports three actions via JSON body: "create", "update", "delete".

**Common auth check** at the top of the handler.

**Action: "create"**
Body: `{ action: "create", projectId, fields: { name, category?, date, endDate?, notes? } }`
- Validate: `name` is non-empty string -> 400 "Event name is required"
- Validate: `date` matches YYYY-MM-DD format -> 400 "Invalid date format"
- If `endDate` provided, validate same format
- If `category` provided, validate against the 10 allowed categories from Phase 15 D-04:
  ```typescript
  const VALID_CATEGORIES = [
    "design", "construction", "delivery", "installation",
    "inspection", "meeting", "review", "deadline", "payment", "other"
  ];
  ```
  Reject invalid category with 400 "Invalid event category"
- Generate `eventKey` via `generatePortalToken(8)`
- Build event object:
  ```typescript
  const event = {
    _key: eventKey,
    _type: "customEvent",
    name: fields.name,
    category: fields.category || "other",
    date: fields.date,
    endDate: fields.endDate || null,
    notes: fields.notes || "",
  };
  ```
- Patch Sanity:
  ```typescript
  await sanityWriteClient
    .patch(projectId)
    .setIfMissing({ customEvents: [] })
    .append("customEvents", [event])
    .commit();
  ```
- Return 200 with `{ success: true, eventKey }`

**Action: "update"**
Body: `{ action: "update", projectId, eventKey, fields: { name?, category?, date?, endDate?, notes? } }`
- Validate: `eventKey` is non-empty string
- Build dynamic set object (same pattern as update-procurement-item edit action):
  ```typescript
  const setObj: Record<string, unknown> = {};
  if (fields) {
    for (const [fieldName, value] of Object.entries(fields)) {
      if (fieldName === "category" && value) {
        if (!VALID_CATEGORIES.includes(value as string)) {
          return new Response(JSON.stringify({ error: "Invalid event category" }), { status: 400, ... });
        }
      }
      setObj[`customEvents[_key=="${eventKey}"].${fieldName}`] = value;
    }
  }
  await sanityWriteClient.patch(projectId).set(setObj).commit();
  ```
- Return 200 with `{ success: true }`

**Action: "delete"**
Body: `{ action: "delete", projectId, eventKey }`
- Validate: `eventKey` is non-empty string
- Patch Sanity:
  ```typescript
  await sanityWriteClient
    .patch(projectId)
    .unset([`customEvents[_key=="${eventKey}"]`])
    .commit();
  ```
- Return 200 with `{ success: true }`

Invalid action -> 400 "Invalid action".

Error handling: try/catch around all Sanity operations, log with `console.error`, return 500.

```typescript
// GROQ: Admin schedule data for the Gantt editor
const ADMIN_SCHEDULE_QUERY = `
  *[_type == "project" && _id == $projectId][0] {
    _id,
    title,
    engagementType,
    isCommercial,
    contractors[]{ ..., contractor->{_id, name, company, trades} },

    customEvents,
    scheduleDependencies
  }

export async function getAdminScheduleData(projectId: string) {
  return sanityClient.fetch(ADMIN_SCHEDULE_QUERY, { projectId });
}
```
