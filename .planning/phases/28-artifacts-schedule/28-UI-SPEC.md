# Phase 28 -- UI Design Contract

> Visual and interaction contract for Artifact Version Manager and Schedule (Gantt) Editor admin pages. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Tailwind v4 tokens in `src/styles/global.css`) |
| Preset | not applicable |
| Component library | none -- hand-rolled Tailwind components following ProcurementEditor patterns |
| Icon library | lucide-react 1.7.0 (Upload, FileText, Trash2, X, Plus, Calendar, Check, CheckCircle, ChevronDown, ChevronUp, Clock, Pencil, MessageSquare, Bell, GripVertical) |
| Font | DM Sans (body, labels, UI), Cormorant Garamond (page headings only) |

**Note:** This phase builds two new admin pages (`/admin/projects/[projectId]/artifacts` and `/admin/projects/[projectId]/schedule`) as SSR Astro pages with React islands. All styling uses Tailwind utility classes referencing the project's custom `@theme` tokens. No shadcn, no external UI library. The Gantt chart uses Frappe Gantt 1.2.2 with existing custom CSS from `gantt.css`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in badges and inline labels (`gap-1`) |
| sm | 8px | Gap between stacked metadata lines, compact list item spacing (`gap-2`, `space-y-2`) |
| md | 16px | Card internal padding, gap between card grid items (`p-4`, `gap-4`) |
| lg | 24px | Card padding on expanded state, section gaps (`p-6`, `gap-6`) |
| xl | 32px | Page-level top margin below breadcrumbs (`mt-8`) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Popover minimum width: 280px (enough for date pickers and form fields)
- Popover maximum width: 360px (prevents popover from overwhelming Gantt chart)
- Upload drop zone minimum height: 80px
- Gantt bar height: 28px (set by existing `--gv-bar-height` in gantt.css)

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm font-body` |
| Label | 12px | 400 (regular) | 1.5 | `text-xs font-body uppercase tracking-widest` |
| Heading | 24px | 400 (normal) | 1.2 | `text-2xl font-heading font-normal` |
| Subheading | 14px | 600 (semibold) | 1.5 | `text-sm font-semibold font-body` |

**Source:** Matches ProcurementEditor established patterns exactly. Page heading uses `font-heading` (Cormorant Garamond). All other text uses `font-body` (DM Sans). Labels are uppercase with widest tracking per the admin convention.

| Dominant (60%) | `#FAF8F5` (cream) | Page background (inherited from AdminLayout body) |

## Interaction Contracts

### Artifact Card Expand/Collapse

1. User clicks anywhere on a collapsed artifact card
2. Card expands in-place to show upload form, version list, and activity log
3. Chevron icon rotates from `ChevronDown` to `ChevronUp`
4. Card padding increases from `p-4` to `p-6`
5. Only one card can be expanded at a time (expanding one collapses the previous)
6. User clicks the card header or the chevron icon to collapse
7. Expand/collapse uses conditional rendering (no animation -- keep it simple)

### Upload New Version

1. User clicks the drop zone or drags a file onto it
2. File input accepts any file type (the schema allows any file)
3. User optionally types a note in the note input
4. User clicks "Upload Version"
5. Button text changes to "Uploading..." with `animate-pulse`
6. API call: `POST /api/admin/artifact-version` with FormData (file, note, projectId, artifactKey)
7. On success: new version appears at top of version list, marked as current
8. On failure: error toast/message below upload form in `text-xs text-red-600 font-body`
9. Button text reverts to "Upload Version"

### Set Version as Current

1. User clicks "Set as Current" button on a non-current version row
2. Button shows loading state (opacity-50, disabled)
3. API call: `POST /api/admin/artifact-version` with action "set-current", projectId, artifactKey, versionKey
4. On success: accent border moves to the newly-current version, previous current loses border
5. Local state updates optimistically

### Add New Artifact

1. User clicks "Add Artifact" button
2. Inline form appears at top of page (above the card grid) with type dropdown and file upload
3. User selects artifact type from dropdown (6 standard types)
4. User uploads first file (required)
5. User clicks "Add" button
6. API call: `POST /api/admin/artifact-crud` with action "add", type, file, projectId
7. On success: new artifact card appears in the grid, form collapses
8. On failure: error message below form

### Remove Artifact

1. User clicks `Trash2` icon in expanded card header
2. Confirmation dialog appears (fixed overlay pattern from ProcurementEditor)
3. User clicks "Remove" to confirm or "Keep Artifact" to cancel
4. API call: `POST /api/admin/artifact-crud` with action "remove", projectId, artifactKey
5. On success: card removed from grid with no animation
6. On failure: error message, dialog stays open

### Gantt Bar Click (Edit)

1. User single-clicks a bar or marker on the Gantt chart
2. If `lastDragTaskId` matches the clicked task -- skip (drag just completed)
3. Parse task `id` format `"category:_key"` to determine item type
4. Popover appears anchored near the bar (positioned via `getBoundingClientRect()` offset)
5. Popover shows appropriate fields for the item type (see popover variants table)
6. User edits fields and clicks "Save"
7. API call: `POST /api/admin/schedule-date` with category, _key, updated fields
8. On success: popover closes, Gantt refreshes with updated data
9. Clicking outside the popover or pressing Escape closes it without saving

### Gantt Bar Drag (Immediate Save)

1. User presses and drags a bar on the Gantt chart
2. Frappe Gantt provides native drag visual feedback
3. On drop, `on_date_change(task, newStart, newEnd)` fires
4. Set `lastDragTaskId` ref to prevent popover opening
5. Immediately `POST /api/admin/schedule-date` with new dates
6. No confirmation, no undo
7. On failure: log error, Gantt will show stale data until page refresh (acceptable for now)

### Gantt Empty Space Click (Create Event)

1. User clicks empty grid space on the Gantt timeline
2. `on_date_click(dateString)` fires with "YYYY-MM-DD" string
3. "New Event" popover appears at the click position
4. Start date pre-filled from the clicked date
5. User fills name (required), optionally category, end date, notes
6. User clicks "Create Event"
7. API call: `POST /api/admin/schedule-event` with action "create", fields
8. On success: popover closes, Gantt refreshes with new event bar
9. Validation: if name is empty on submit, name input gets `border-red-400` and focus

### Popover Dismiss

1. Click outside popover: closes without saving
2. Press Escape: closes without saving
3. Click close (X) button: closes without saving
4. These all reset popover state to null
