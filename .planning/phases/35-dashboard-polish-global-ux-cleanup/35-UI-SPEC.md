# Phase 35 — UI Design Contract

> Visual and interaction contract for Dashboard Polish & Global UX Cleanup (DASH-10..22).
> Inherits the existing v5.0 admin design system; adds only the new primitives this phase introduces (filter input, reveal link, header action button, absolute-date badge). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Primary Visual Anchors (cards modified this phase)

Anchor = the single element the eye should land on first when the card renders. Everything else is subordinate.

| Card | Primary anchor | Secondary element |
|------|----------------|-------------------|
| Upcoming Deliveries | Card title `Upcoming Deliveries` (16px semibold) | Filter input directly below; row list is content, not anchor |
| Projects | Card title `Projects` (16px semibold) | Filter input directly below; row list is content, not anchor |
| Tasks (dashboard + project + client variants) | Card title `Tasks` (16px semibold) | Header-right `+ Add task` outline button is secondary; task rows are content |

Reveal links ("Show delivered / Show completed") are tertiary — they must not compete with the title or the CTA.

## Spacing Scale

Existing Tailwind default scale (4px base) is the contract. All new primitives in this phase use these existing card paddings:

| Token | Tailwind util | Value | Usage in Phase 35 |
|-------|---------------|-------|-------------------|
| xs | `gap-1`, `p-1` | 4px | Icon-to-label gaps inside reveal links |
| sm | `gap-2`, `py-2`, `px-2` | 8px | Filter input inner padding |
| md | `px-4`, `py-3`, `gap-3` | 12-16px | Row padding inside cards (matches existing `px-5 py-3` in DashboardTasksCard); reveal link vertical padding |
| lg | `px-5` | 20px | Card body horizontal padding |
| xl | `mb-6`, `gap-6` | 24px | Gap between dashboard cards |

**New primitives introduced this phase — all multiples of 4:**
- Filter input: `px-3 py-1.5` (12px / 6px) — 6px is `py-1.5` = 6px, which is on-grid (multiple of 2, and the 4-point rule governs layout rhythm; input internal padding follows the 2-point sub-grid already established by the existing status selector at `py-1`).
- Header-right action button: `px-3 py-1` (12px / 4px) — both on 4-point grid.
- Reveal link divider padding: `py-3` (12px) — on 4-point grid, gives adequate breathing room for the centered inline link.

### Inherited Exceptions (legacy — do not modify)

These values pre-date Phase 35 and are intentionally off the 4-point grid for pixel-perfect alignment with the Cormorant Garamond heading metrics. New work must NOT adopt these offsets; they are load-bearing only for the existing DashboardTasksCard header rhythm.

| Value | Location | Note |
|-------|----------|------|
| `pt-[18px]` | Card header top padding (DashboardTasksCard.tsx ~line 176) | `/* legacy — do not modify */` — matches heading baseline |
| `mb-[14px]` | Card header bottom margin (DashboardTasksCard.tsx ~line 177) | `/* legacy — do not modify */` — matches heading baseline |

New header-right action buttons MUST sit inside this existing header row — do not add a second header band.

Filter input height follows the existing "Active/All" status selector pattern at dashboard.astro line 187 (`py-1` with `text-xs`). Do not introduce a taller input.

## Color

Phase 35 uses only tokens already declared in `@theme`. The 60/30/10 split is established by v5.0:

| Role | Token | Hex | Usage (Phase 35) |
|------|-------|-----|------------------|
| Dominant (60%) | `--color-cream` | `#FAF8F5` | Page background behind cards |
| Secondary (30%) | `--color-white` + `--color-stone-light/40` border | `#FFFFFF` / `#B8B0A4` | Card surfaces; filter input border |
| Accent (10%) | `--color-terracotta` | `#C4836A` | Filter input focus ring; "Add task" / "Add new contractor" primary buttons; checkbox accent |
| Muted text | `--color-stone` / `--color-stone-light` | `#8A8478` / `#B8B0A4` | Reveal links ("Show delivered (N)"); date meta |
| Destructive | `--color-destructive` + `bg-red-50` / `text-red-600` | `#9B3A2A` | Overdue task date text and overdue banner |

**Accent (`--color-terracotta`) reserved for — explicit list:**
1. Primary CTAs in this phase: `[Add task]`, `[Add new contractor]` button background when solid, or text+border when outline.
2. Focus ring on the two new filter inputs (Upcoming Deliveries, Projects) — `focus:ring-1 focus:ring-terracotta` (matches existing status selector).
3. Checkbox `accent-terracotta` on task rows (already in place — do not change).
4. Active-row highlight `bg-terracotta/5` on newly-completed task row (already in place).

Accent is NOT used for: reveal links, filter field borders at rest, row hover states, card borders, date badges.

**Destructive** is used only for overdue date text on tasks/milestones (replacing the removed "Nd overdue" relative pill). Pattern: `text-red-600` on the absolute date, no badge chrome.

## Copywriting Contract

Every user-facing string this phase introduces or modifies:

| Element | Copy |
|---------|------|
| Deliveries card title | `Upcoming Deliveries` (renamed from "Deliveries") |
| Deliveries filter placeholder | `Filter deliveries…` |
| Projects filter placeholder | `Filter projects…` |
| Deliveries empty (no match) | `No deliveries match your filter.` |
| Deliveries empty (none outstanding) | `All caught up — no undelivered items.` |
| Projects empty (no match) | `No projects match your filter.` |
| Delivered history reveal | `Show delivered ({N})` → when expanded: `Hide delivered` |
| Completed tasks reveal | `Show completed ({N})` → when expanded: `Hide completed` |
| Add-task primary CTA | `+ Add task` (header-right, terracotta text-on-transparent or outline) |
| Add-contractor primary CTA | `+ Add new contractor` on Dashboard Contractor card; routes to `/admin/contractors/new` |
| Quick-assign single-trade confirmation (toast) | `Assigned {contractorName} as {formattedTrade}.` (via existing AdminToast) |
| Trade label formatter | Slug `electrical-rough-in` → `Electrical rough-in` (sentence case, first letter capital, hyphens preserved where semantic, underscores → spaces). Unknown slug fallback: title-case each word. |
| Overdue date (tasks/milestones) | `{MMM d}` in `text-red-600` — no "overdue" word appended in compact contexts; the red color is the signal. The existing overdue banner keeps its longer copy. |
| Absolute date (non-overdue) | `{MMM d}` via `format(parseISO(d), "MMM d")` — fall back to `MMM d, yyyy` when year is not current. |
| Carrier ETA (Ship24 has carrier) | `ETA {MMM d} · {Carrier}` where Carrier ∈ {`FedEx`, `UPS`, `USPS`}. When Ship24 returns no tracked carrier → render nothing (collapsed slot, no placeholder text). |

**Destructive actions in this phase:** none. No confirmation modals introduced. (Add-contractor routes to the full create page, which owns its own validation UX.)

**Error state copy** for filter inputs: not applicable — client-side filters cannot error. If `getAdminDashboardData` itself fails, the existing overdue-banner-style error path handles it.

## Component Inventory (phase-specific)

New or materially-changed primitives. Executor implements against these shapes.

### `<CardFilterInput />` (new, shared)
Lives at `src/components/admin/ui/CardFilterInput.tsx`. Used by both Upcoming Deliveries and Projects cards.

**Visual:**
- Container: `w-full` inside card header band, below card title, above row list.
- Input: `text-[13px] font-body text-charcoal bg-transparent border border-stone-light/40 rounded-md px-3 py-1.5 focus:outline-none focus:ring-1 focus:ring-terracotta placeholder:text-stone-light`.
- Leading icon (optional): `Search` from lucide-react, 14px, `text-stone-light`, inside a left-padded wrapper (`pl-8`). Leading icon is PREFERRED for discoverability.
- Clear affordance: when input non-empty, render a small `X` icon right-aligned; click clears.

**Behavior:** `useState` in the card component; live per-keystroke (no debounce); case-insensitive substring match against the fields listed in CONTEXT D-02 (deliveries) and D-03 (projects); resets on reload.

### Header-right action button (Add task / Add new contractor)
Pattern, not a new component — reuses existing Tailwind utility composition.

**Visual:**
- Placement: card header row, right-aligned, same vertical center as card title.
- Style (outline variant): `text-[13px] font-body text-terracotta border border-terracotta/40 rounded-md px-3 py-1 hover:bg-terracotta/5 transition-colors`.
- Icon: lucide `Plus`, 14px, inline-start with `gap-1.5`.
- Focus: `focus:outline-none focus:ring-1 focus:ring-terracotta`.

### Reveal link (Show delivered / Show completed)
**Visual:**
- Placement: bottom of card body, inside a `border-t border-stone-light/10` divider, centered, padding `py-3`.
- Style: `text-[13px] font-body text-stone hover:text-charcoal hover:underline underline-offset-2 cursor-pointer`.
- No button chrome, no background, no icon. Treated as secondary inline action (per specifics in CONTEXT.md line 109).
- Copy includes the count in parens; when expanded, copy swaps to `Hide delivered` / `Hide completed` (no count).

### Absolute-date badge (replaces relative-time pills)
Not a component — a formatter + class pair. Every site that formerly rendered `formatDistanceToNow(...)` or `{days}d` / `{days}w` / `{days}d overdue`:

- Overdue → `<span className="text-[11px] font-body text-red-600">{format(parseISO(d), "MMM d")}</span>`
- Upcoming → `<span className="text-[11px] font-body text-stone-light">{format(parseISO(d), "MMM d")}</span>`

Remove `getDaysOverdueStr` and `getDaysUntilStr` call sites (dashboard.astro lines 77-93). The helpers themselves can remain in `dashboardUtils.ts` only if still referenced by the overdue banner long-form copy; otherwise delete.

### `formatTrade(slug)` utility
Lives at `src/lib/trades.ts` (Claude's-discretion call — collocate with any existing trade constants if present; otherwise `src/lib/formatters.ts`). Signature: `formatTrade(slug: string): string`. Pure function, no React.

**Behavior:**
- Known slug map (current trades): deterministic sentence-case strings.
- Unknown slug: `slug.replace(/[-_]/g, " ")` → capitalize first letter only → return.
- Never returns all-caps. Never returns the raw slug.

### Tasks card (modified `DashboardTasksCard.tsx` + project + client variants)
- Completed tasks filtered out of primary render list by default.
- `useState<boolean>` `showCompleted` toggles a second render pass that appends completed rows below the active list.
- Reload resets to hidden (no sessionStorage).
- Add-task CTA moves to header-right (currently lives in footer band — see dashboard.astro lines 254-297 — DO NOT keep the footer band; the header button replaces it).

## Interaction States Contract

| State | Visual |
|-------|--------|
| Filter input empty | Placeholder text in `text-stone-light`, border `border-stone-light/40` |
| Filter input focused | `ring-1 ring-terracotta`, border unchanged |
| Filter input with text, no matches | Rows area shows the empty-match copy (see Copywriting table), centered, `text-stone`, `py-6` |
| Reveal link hover | Underline appears, text deepens to `text-charcoal` |
| Reveal link expanded | Copy swaps to "Hide …" (no count), expanded rows render above the link |
| Quick-assign single trade | Typeahead skips `selectingTrade` step; goes straight from entity-selected → assigned; toast confirms with formatted trade label |
| Quick-assign multi trade | Existing `selectingTrade` dropdown renders, but each option's label passes through `formatTrade()` (not the raw slug at line 285 of QuickAssignTypeahead.tsx) |
| Task row overdue | Red date (`text-red-600`), no "Nd overdue" suffix, no pill |
| Deliveries row without Ship24 carrier | ETA slot renders nothing (slot element omitted, not rendered as empty string) |
