| Astro.locals.tenantId -> getTenantClient | Tenant ID from middleware flows to Sanity client factory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-10 | Elevation of Privilege | Admin API routes tenant isolation | mitigate | Every admin API route extracts tenantId from session (set at login, not user-controllable); getTenantClient validates tenantId exists in config; unknown tenantId throws Error |
| T-29-11 | Tampering | GROQ query injection via tenant config | accept | Tenant config is a static JSON file in the repo, not user-editable at runtime; dataset names are validated by Sanity API |
| T-29-12 | Information Disclosure | Cross-tenant data leakage | mitigate | Each tenant gets separate Sanity dataset + write token; getTenantClient only returns client for the session's tenantId; no global client used in admin code after refactor |
| T-29-13 | Repudiation | Who modified admin data | accept | Sanity Content Lake has built-in revision history; admin session identifies the user (entityId in session); sufficient for 2-user system |
</threat_model>
