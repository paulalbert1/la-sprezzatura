    **Extend `src/middleware.ts`** -- add admin route block BEFORE the final `return next()`. The admin block must:
    1. Check if `pathname.startsWith("/admin")` or `pathname.startsWith("/api/admin")`
    2. Skip if path is `/admin/login` or `/api/admin/login` (public paths)
    3. Call `getSession(context.cookies)`
    4. If no session, or `session.role !== "admin"`, or `!session.tenantId` -- redirect to `/admin/login`
    5. Set `context.locals.tenantId = session.tenantId`
    6. Set `context.locals.role = session.role`
    7. Call `next()`

    The exact code to add BEFORE the final `return next()` in the `onRequest` handler:
    ```typescript
    // Admin routes (pages and API)
    if (pathname.startsWith("/admin") || pathname.startsWith("/api/admin")) {
      if (pathname === "/admin/login" || pathname === "/api/admin/login") return next();

      const session = await getSession(context.cookies);
      if (!session || session.role !== "admin" || !session.tenantId) {
        return context.redirect("/admin/login");
      }

      context.locals.tenantId = session.tenantId;
      context.locals.role = session.role;
      return next();
    }
    ```

    CRITICAL: Do NOT modify the existing portal, workorder, or building blocks. Only ADD the admin block.

| Browser -> /admin/* middleware | Unauthenticated requests to protected admin routes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-06 | Spoofing | /admin/* middleware | mitigate | Session check requires role=admin AND tenantId present; missing either redirects to login |
| T-29-07 | Elevation of Privilege | /api/admin/* without middleware | mitigate | API routes also independently check session (defense in depth); middleware covers pages, route-level checks cover API |
| T-29-08 | Information Disclosure | Admin login form | mitigate | noindex,nofollow meta tag; no admin data visible until authenticated |
| T-29-09 | Denial of Service | Admin login form rapid submissions | mitigate | Button disabled during submission; server-side rate limit in Plan 01 (5 per 15min per IP) |
</threat_model>
