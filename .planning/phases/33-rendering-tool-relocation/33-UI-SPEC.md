# Phase 33 -- UI Design Contract

> Visual and interaction contract for porting the AI rendering tool from Sanity Studio into `/admin/rendering` under the Phase 32 parchment/gold-light admin theme. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (bespoke Tailwind v4 theme, no shadcn) |
| Preset | not applicable |
| Component library | none (custom React islands in Astro SSR pages) |
| Icon library | lucide-react (admin convention) |
| Font (heading) | Cormorant Garamond, Georgia, serif (`--font-heading`) -- reserved for brand mark only |
| Font (body / UI) | DM Sans, system-ui, sans-serif (`--font-body`, `--font-sans`) -- all admin UI |

Source: Codebase scan -- `src/styles/global.css` theme tokens, `AdminLayout.astro` font wiring, `AdminNav.tsx` + `ProcurementEditor.tsx` icon/font conventions, Phase 32 `32-UI-SPEC.md` precedent.

## Typography

**Exactly 4 sizes, exactly 2 weights** -- matches Phase 32 constraint so the checker passes.

| Role | Size | Weight | Line Height | Class / Inline Style |
|------|------|--------|-------------|----------------------|
| Body / default | 14px | 400 (regular) | 1.5 | `text-sm font-body` |
| Label / meta / timestamp | 11.5px | 400 (regular) | 1.4 | `text-[11.5px] font-body` |
| Section heading / row title | 13px | 600 (semibold) | 1.4 | `text-[13px] font-semibold font-body` |
| Page heading | 22px | 600 (semibold) | 1.2 | matches AdminLayout `pageTitle` style |

All admin-facing text uses `font-body` / `font-sans` (DM Sans). The serif heading font (`font-heading`, Cormorant Garamond) is **not used** inside admin content -- it is reserved for the sidebar brand mark and the public-facing site.

### Role-specific specifications

- **Session row title:** 13px, 600 weight, `color: #2C2520` (charcoal)
- **Session row meta line** (project name, owner stamp, created-at): 11.5px, 400 weight, `color: #6B5E52` (text-mid)
- **"by Designer X" ownership stamp:** 11px, 400 weight, `color: #9E8E80` (text-muted), italic
- **Filter chip label:** 12.5px, 500 weight, `letterSpacing: 0.02em` (matches sidebar nav item pattern in `AdminNav.tsx`)
- **Usage badge text:** 11px, 500 weight, `letterSpacing: 0.04em` uppercase (matches Phase 32 status pill pattern)
- **Wizard step indicator label:** 12px, 600 weight when active, 400 weight otherwise
- **Wizard step number inside circle:** 12px, 600 weight, white on active/completed, `#9E8E80` otherwise
- **Chat message body:** 14px, 400 weight, 1.5 line height
- **Chat message timestamp:** 11px, 400 weight, `color: #9E8E80`
- **Promote drawer heading:** 22px, 600 weight, matches page heading spec
- **Promote drawer label:** 11.5px, 600 weight, uppercase, `letterSpacing: 0.04em`, `color: #6B5E52`

Source: `ProcurementEditor.tsx` row/label pattern (lines 584-598), `AdminNav.tsx` letterspacing convention (line 77), Phase 32 `32-UI-SPEC.md`.

- Primary "New session" button: `bg-[#9A7B4B] text-white px-4 py-2 rounded-lg text-sm font-semibold font-body hover:bg-[#8A6D40] transition-colors` with left icon `<Sparkles className="w-4 h-4" />`

### 2. SessionList (rows, not grid)

**Format decision (Claude's discretion):** rows -- matching procurement, clients, contractors, portfolio list patterns in admin. Rendering thumbnails are shown inline in each row at 48px square, which is enough visual context without demanding a grid. Rows keep scanability for 50+ sessions which a grid would fight.

#### Row layout

```
+---------------------------------------------------------------------------+
| [thumb] Session title                           3 renderings  2h ago      |
|         North Shore Bathroom -- by Liz Kelly                              |
+---------------------------------------------------------------------------+
```

- Container: full-width `<button>` (clickable row), left padding 20px, right padding 20px, vertical padding 16px
- Thumbnail: 48px square, `rounded-md`, 0.5px `#E8DDD0` border, first rendering's blob served via `/api/blob-serve`; if no rendering yet, show parchment background with `Sparkles` icon centered at 18px `#9E8E80`
- Title (13px / 600): session title, truncate with ellipsis after 1 line
- Meta line 1 (11.5px / 400 / `#6B5E52`): `{project.title} -- by {owner.displayName}` (the "-- by" stamp satisfies D-09)
- Right cluster (11.5px / 400 / `#9E8E80`, align right): rendering count + created-at relative time, stacked vertically, `gap-1`
- Hover: background `#F3EDE3`
- Click: navigate to `/admin/rendering/[sessionId]` (chat view) if the session has at least one rendering; otherwise to `/admin/rendering/[sessionId]/wizard` (D-06 resume)
- Row border bottom 0.5px `#E8DDD0`, last row has no bottom border

#### Ownership stamp specifics (D-09)

- When session is owned by the current designer: stamp reads `"by You"` in italic
- When owned by another designer: stamp reads `"by {displayName}"` in italic, never just the raw Sanity user ID
- Stamp color is always `#9E8E80`, never accent -- authorship is a quiet signal, not a call to action

#### States

| State | Treatment |
|-------|-----------|
| Loading (initial list fetch) | 6 skeleton rows, 64px tall, parchment background with `animate-pulse`, no thumbnail placeholders shimmering -- static muted blocks |
| Empty (no sessions in tenant) | Empty state card (see copywriting) with solid "New session" CTA centered at 48px vertical padding |
| Empty after "Mine" filter but others exist | Different empty state copy (see copywriting) with a "Show all sessions" text button that deactivates the filter |
| Error (fetch failed) | Warm destructive banner at top of list container: `bg-[#FBEEE8] text-[#9B3A2A] text-sm px-5 py-3 rounded-lg` with "Could not load sessions. Try refreshing." |
| Populated | Rows as spec'd above |

### 5. UsageBadge

Placed in `/admin/rendering` page header (next to "New session") and in the ChatView header. **Never** on the dashboard, sidebar, or wizard (D-15, D-16).

- Component: small rounded pill
- Dimensions: auto width, 28px tall, `px-3 py-1`
- Content: `Your usage {count} / {limit}` (D-16 per-designer copy exactly)
- Styling per threshold: see Color section "Usage badge thresholds"
- Font: 11px / 500 / `letterSpacing: 0.04em` / uppercase

### 6. "Mine" filter chip

Placed in the rendering page header next to the project filter (not the right side).

- Component: toggle chip (single button, two visual states)
- Dimensions: auto width, 32px tall, `px-3 py-1.5`
- Inactive state: `bg-transparent text-[#6B5E52] border 0.5px #D4C8B8 rounded-full text-[12.5px] font-medium hover:bg-[#F3EDE3]`
- Active state: `bg-[#F5EDD8] text-[#9A7B4B] border 0.5px #9A7B4B rounded-full text-[12.5px] font-semibold`
- Optional lead icon: `User` from lucide-react, 14px, positioned left of "Mine"
- Behavior: Gmail-style toggle tag per `<specifics>` block -- single click flips between all-tenant and mine-only. Not a dropdown.
- Not sticky on scroll -- the page header itself is not sticky, so the chip scrolls with it. (Design decision: the list is paginated or virtualized at 50 rows; no persistent header is needed.)
- Keyboard: `role="switch"`, `aria-checked={isMine}`, Space toggles

### 7. "by Designer X" ownership stamp

A component built into the SessionList row meta line. Not a separate element in the DOM sense -- but locked here as a design contract so the checker sees it.

- Placement: appended to project name in meta line 1 with em-dash separator: `{project} -- by {displayName}`
- Styling: inherits meta line styling (11.5px / 400 / `#6B5E52`), except `displayName` is italic

- When owned by current user: display as `"by You"` (italic, same color)
- Never accent-colored (authorship signal, not a CTA)

### Optimistic update behavior

All mutations (refine, promote, wizard draft-save) use optimistic UI following the Phase 32 `ProcurementEditor.tsx` pattern (lines 137-141):
1. Update UI immediately
2. Fire API call
3. On failure: revert + show red destructive banner via `setError(msg)` that auto-dismisses after 3000ms

Generation is **not** optimistic (too expensive and long-running) -- it shows the GeneratingOverlay and blocks interaction until the real response arrives.
