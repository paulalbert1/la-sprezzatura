# Phase 22 — UI Design Contract

> Visual and interaction contract for Procurement Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Sanity Studio environment) |
| Preset | not applicable |
| Component library | @sanity/ui (bundled with sanity v5) |
| Icon library | @sanity/icons (bundled) |
| Font | Sanity Studio default (Inter via @sanity/ui) |

**Context:** This phase operates entirely within Sanity Studio. All UI components use `@sanity/ui` primitives (Card, Text, Stack, Flex, Button, Badge, Spinner, TextInput). The visual environment is governed by `studioTheme.ts` (custom `buildTheme` palette) and `studio.css` (CSS variable overrides). No shadcn, no Tailwind, no custom CSS framework.

## Spacing Scale

Sanity Studio uses `@sanity/ui` spacing props (integer scale 1-5 mapped to rem values). All components in this phase use the `@sanity/ui` spacing system directly.

| Token | @sanity/ui prop | Approx px | Usage in this phase |
|-------|----------------|-----------|---------------------|
| 1 | `space={1}` / `padding={1}` | 4px | Inline icon gaps |
| 2 | `space={2}` / `padding={2}` | 8px | Compact element spacing (Flex gaps, Stack space between filename and remove button) |
| 3 | `space={3}` / `padding={3}` | 12px | Default Card padding (NetPriceDisplay card, error states, file preview cards) |
| 4 | `space={4}` / `padding={4}` | 16px | Section-level spacing (files array wrapper) |
| 5 | `space={5}` / `padding={5}` | 20px | Not used this phase |

Exceptions: none. All spacing uses `@sanity/ui` integer props.

## Typography

All text uses `@sanity/ui` `<Text>` component with `size` prop (integer scale 0-4). No custom font declarations.

| Role | Component | Size Prop | Weight | Line Height | Usage |
|------|-----------|-----------|--------|-------------|-------|
| Body | `<Text size={1}>` | 1 | regular (400) | 1.5 (inherited) | File labels, upload help text, muted descriptions |
| Label | `<Text size={1} weight="medium">` | 1 | medium (500) | 1.5 (inherited) | Field descriptions (e.g., "e.g., COM form -- Kravet") |
| Computed value | `<Text size={2} weight="medium">` | 2 | medium (500) | 1.5 (inherited) | Net price display: "Net: $450.00" |
| Muted guidance | `<Text size={1} muted>` | 1 | regular (400) | 1.5 (inherited) | Empty state: "Enter retail price and client cost to see net price" |

Font weights used: 400 (regular) and 500 (medium). No bold (700) or semibold (600) in this phase's custom components. Sanity Studio's own field titles use its built-in weight system.

### CSS Badge Fix (D-10)

The `[data-ui="Badge"]` selector is being REMOVED from the override rule on line 81-86 of `studio.css`. After removal, Badge components will render their native tone colors instead of being forced to warm gray (`#F0EEEB` bg, `#0F172A` fg). The remaining selectors (`[data-ui="StatusButton"]`, `[data-testid="review-changes-button"]`) keep their warm gray override.

**Post-fix tone rendering:** Badge components with `tone="positive"` will show green, `tone="caution"` will show yellow, `tone="primary"` will show amber, `tone="critical"` will show red -- all inheriting from `studioTheme.ts` palette.

Accent reserved for: CTA buttons (Publish, Save), active navigation tabs, accent links. NOT used on procurement badges (badges use tone system).

## Interaction Contracts

### NetPriceDisplay

| State | Trigger | Visual |
|-------|---------|--------|
| Empty | Both retailPrice and clientCost are undefined/0 | `<Card padding={3} radius={2} tone="default">` with `<Text size={1} muted>`: "Enter retail price and client cost to see net price" |
| Computed | Either price field has a value | `<Card padding={3} radius={2} tone="default">` with `<Text size={2} weight="medium">`: "Net: $X.XX" |
| Live update | User changes retailPrice or clientCost | Value recalculates instantly via `useFormValue` reactivity -- no save required |
| Negative result | clientCost > retailPrice | Display negative: "Net: -$50.00" (no special styling -- informational only) |
| Zero result | Both fields equal | Display: "Net: $0.00" |

**Format:** Dollar sign prefix, two decimal places, converted from cents. `(retailPrice - clientCost) / 100` with `.toFixed(2)`.

### BlobFileInput (Enhanced)

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (no file) | Initial render, after remove | "Choose File" ghost button + muted help text: "Upload a PDF, JPEG, PNG, or WebP file" |
| Uploading | File selected | Spinner + "Uploading..." text |
| Error | Upload fails | Critical-tone Card: "Upload failed: {message}" + "Try Again" ghost button |
| Uploaded (image) | JPEG/PNG/WebP file uploaded | 48px square thumbnail (`<img>` with `border-radius: 4px`, `object-fit: cover`) + truncated filename (40 char max) + "Remove" critical ghost button |
| Uploaded (document) | PDF or other file uploaded | DocumentIcon (from @sanity/icons, 24px) + truncated filename + "Remove" critical ghost button |

**Thumbnail URL construction:** Use `/api/blob-serve?path={pathname}&source=studio&token={STUDIO_API_SECRET}` pattern from existing blob-serve route. For image detection, check file extension: `.jpg`, `.jpeg`, `.png`, `.webp` are images; everything else shows the document icon.

**Click behavior on thumbnail/icon:** Opens the file in a new tab via the blob-serve URL (`window.open`).

### Files Array (Schema-Level)

| State | Trigger | Visual |
|-------|---------|--------|
| Empty array | No files added | Sanity's default "Add item" button for arrays |
| Array with items | Files added | Each item shows in Sanity's array list with preview: title = label ("COM form -- Kravet"), subtitle = filename extracted from path |
| Add new item | Click "Add item" | Opens inline object form with Label (text input) and File (BlobFileInput) fields |

## Field Ordering Contract

New fields are added to the procurement item form in this order (Claude's discretion per CONTEXT.md):

| Position | Field | Type | Rationale |
|----------|-------|------|-----------|
| 1 | name | string (existing) | Identity -- what is this item |
| 2 | manufacturer | string (NEW) | Identity -- who makes it |
| 3 | status | string/options (existing, renamed) | Current state in pipeline |
| 4 | quantity | number (NEW) | How many |
| 5 | retailPrice | number (existing) | Price info block starts |
| 6 | clientCost | number (existing) | Price info block continues |
| 7 | netPrice | number/readOnly (NEW, computed) | Price info block ends with computed result |
| 8 | orderDate | date (existing) | Date info block starts |
| 9 | expectedDeliveryDate | date (existing) | Date info block continues |
| 10 | installDate | date (existing) | Date info block ends |
| 11 | trackingNumber | string (existing) | Logistics metadata |
| 12 | files | array (NEW) | Attachments section |
| 13 | notes | text (NEW) | Free-form notes last (longest field, least structured) |

Rationale: Group by purpose (identity, status, pricing, dates, logistics, attachments, notes). New fields slot into their natural groups rather than appending at the end.
