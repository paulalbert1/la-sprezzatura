<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser -> API routes | Untrusted input (projectId, itemKey, status, file) crosses trust boundary |
| API routes -> Sanity Content Lake | Write operations require valid session |
| API routes -> Vercel Blob | File upload/deletion requires server-side token |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-27-03 | Elevation of Privilege | All 3 API routes | high | mitigate | Every route checks `getSession(cookies)` and rejects with 401 if `!session \|\| session.role !== "admin"`. Test file verifies this for each route. |
| T-27-04 | Tampering | update-procurement-status | medium | mitigate | Status value validated against `PROCUREMENT_STAGES.some(s => s.value === status)` before mutation. Invalid values return 400. |
| T-27-05 | Tampering | upload-procurement-file | high | mitigate | File type validated against allowlist `["application/pdf", "image/jpeg", "image/png", "image/webp"]` server-side. Disallowed types return 400. `addRandomSuffix: true` prevents path traversal. |
| T-27-06 | Denial of Service | upload-procurement-file | medium | accept | Vercel Blob has its own size limits. No additional rate limiting added for single-admin context. |
| T-27-07 | Information Disclosure | sanityWriteClient token | high | mitigate | Write client imported only in server-side API route files (never in React components). Per D-15, the React island uses fetch() to API routes only. |
| T-27-08 | Spoofing | itemKey parameter | medium | mitigate | itemKey is used in parameterized Sanity path expressions (not string concatenation into GROQ). Sanity validates _key format internally. |
</threat_model>
