    **Handler functions (8 total):**

    1. `handleStatusChange(itemKey, newStatus)` per D-01/D-03:
       - Save previous status for rollback
       - Optimistic: update items state immediately
       - Close dropdown
       - `fetch('/api/admin/update-procurement-status', { method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify({ projectId, itemKey, status: newStatus }) })`
       - On error: revert to previous status, setMessage({ type: "error", text: "Status update failed. Please try again." })

    2. `openAddPanel()` per D-07:
       - setEditingItem(null)
       - setFormData({ status: "not-yet-ordered" })
       - setPanelOpen(true)

    3. `openEditPanel(item)` per D-07:
       - setEditingItem(item)
       - setFormData with all item fields; convert retailPrice and clientCost from integer cents to dollars for display: `item.retailPrice ? item.retailPrice / 100 : ""`; same for clientCost
       - Copy item.files array into formData.files
       - setPanelOpen(true)

    4. `closePanel()`:
       - setPanelOpen(false), setEditingItem(null), setFormData({})

    5. `handleSaveItem()` per D-08/D-10:
       - setSaving(true), clear message
       - Build fields object from formData: convert retailPrice/clientCost from dollar string to integer cents via `Math.round(parseFloat(value) * 100)` (skip if empty/NaN); convert quantity to number; pass strings through for dates and text fields
       - If editingItem: POST action="edit" with itemKey=editingItem._key and fields
       - If no editingItem: POST action="add" with fields (no itemKey)
       - On success for add: response.json() returns `{ success, item }` with generated _key; prepend/append to local items state; show "Item added"
       - On success for edit: update the matching item in local items state; show "Changes saved"
       - On error: show "Unable to save. Please try again." (inside panel)
       - closePanel() on success
       - setSaving(false)

    6. `handleRemoveItem(item)` per D-09:
       - POST to /api/admin/update-procurement-item with action="remove", projectId, itemKey=item._key
       - On success: filter item from local items state, show "Item removed", setConfirmRemoveItem(null)
       - On error: show error message, setConfirmRemoveItem(null)

    7. `handleFileUpload(files: FileList)` per D-11/D-12:
       - Only works when editingItem is set (need _key for Sanity path)
       - setUploadingFile(true)
       - For each file in FileList: create FormData, append "file", "projectId", "itemKey" (editingItem._key)
       - POST to /api/admin/upload-procurement-file
       - On success: response returns { url, pathname, fileKey }; add { _key: fileKey, label: "", file: url } to formData.files array
       - On error: setMessage({ type: "error", text: "File upload failed. Please try again." })
       - setUploadingFile(false)

    8. `handleFileDelete(fileKey, blobUrl)` per D-13:
       - DELETE to /api/admin/upload-procurement-file with body { projectId, itemKey: editingItem._key, fileKey, blobUrl }
       - On success: remove from formData.files array
       - On error: show error message
