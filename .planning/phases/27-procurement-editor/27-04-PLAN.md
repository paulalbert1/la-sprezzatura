<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Visual and functional verification of procurement editor</name>
  <action>
    This is a human verification checkpoint. No code changes needed. The user verifies the complete procurement editor built in Plans 01-03.

    What was built:
    - Complete procurement editor at /admin/projects/[projectId]/procurement
    - Table with procurement items showing status badges, expected delivery dates, carrier icon links, and overflow menus
    - Inline status dropdown on badge click (D-01/D-02/D-03)
    - Slide-out panel for adding new items and editing existing items with 13 form fields (D-07/D-08)
    - File upload via drag-and-drop or file picker in edit mode (D-11/D-12)
    - File deletion (D-13)
    - Item removal with confirmation dialog (D-09)
    - Overdue date detection with red text (D-05)
    - Carrier icon links parsed from tracking URL domain (D-06)
    - Breadcrumbs: Projects > [Project Name] > Procurement (D-16)
    - Empty state when no items
  </action>
  <verify>
    How to verify:

    1. Start the dev server if not running: `npm run dev`
    2. Navigate to `/admin/login` and log in if needed
    3. Navigate to `/admin/projects` and click any project with procurement items
    4. Click the "Procurement" card on the project overview page
    5. Verify the procurement page loads with breadcrumbs showing: Projects > [Project Name] > Procurement

    Table verification:
    6. Confirm the table shows columns: Item, Status, Delivery, Carrier, Actions
    7. Confirm item names are bold, manufacturer/quantity in smaller gray text below
    8. Confirm status badges use colored pill styling (not plain text)
    9. If any item has a past expected delivery date and is not delivered/installed, confirm the date text is red

    Status dropdown:
    10. Click a status badge -- dropdown should appear below with all 6 stages
    11. Select a different stage -- badge should update immediately
    12. Click outside the dropdown -- it should close

    Slide-out panel (Add):
    13. Click "Add Item" button (top right, terracotta color)
    14. Slide-out panel should animate in from the right
    15. Confirm all 13 form fields are present in order: Name, Manufacturer, Status, Quantity, Retail Price, Client Cost, Order Date, Expected Delivery Date, Install Date, Tracking Number, Tracking URL, Files (shows "Save the item first" hint), Notes
    16. Enter a name and click "Add Item" button in panel footer
    17. Panel should close, new item should appear in table, "Item added" message should show briefly

    Slide-out panel (Edit):
    18. Click the three-dot menu on any item, then click "Edit"
    19. Panel should open pre-populated with item data
    20. If the item has files, the file upload zone should be visible with existing files listed
    21. Make a change, click "Save Changes" -- panel closes, table updates

    File upload (if editing an item):
    22. Open edit panel for any item
    23. Click the file drop zone or drag a PDF/image onto it
    24. File should appear in the list below the drop zone
    25. Click the trash icon on a file to delete it

    Item removal:
    26. Click the three-dot menu on any item, then click "Remove"
    27. Confirmation dialog should appear: "Remove [Item Name]?"
    28. Click "Keep Item" -- dialog closes, item remains
    29. Repeat and click "Remove" -- item should disappear from table

    Empty state:
    30. If no items exist (or after removing all), confirm the empty state text appears: "No procurement items yet" with "Add Item" button
  </verify>
  <done>User types "approved" confirming all interactive elements work correctly, or describes specific issues that need fixing. If issues are found, they are addressed via targeted fixes to Plans 01-03, then this checkpoint is re-run before phase completion.</done>
</task>
