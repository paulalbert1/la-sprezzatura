# Phase 27 — UI Design Contract

> Visual and interaction contract for the procurement editor admin page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro + React islands with Tailwind v4) |
| Preset | not applicable |
| Component library | none (custom React components with Tailwind utility classes) |
| Icon library | lucide-react 1.7.0 (`Plus`, `X`, `Pencil`, `Trash2`, `MoreVertical`, `Upload`, `FileText`, `ExternalLink`, `Package`, `Truck`) |
| Font | Cormorant Garamond (heading via `font-heading`), DM Sans (body via `font-body`) |

**Note:** This phase builds a custom admin page at `/admin/projects/[projectId]/procurement` using the established AdminLayout + React island pattern from Phases 25-26. No shadcn, no component library. All UI is Tailwind utility classes on native HTML elements and React components.

### Table Layout (D-04)

```
+----------------------------------------------+

+----------------------------------------------+

```

**Columns:**

| Column | Width | Alignment | Content |
|--------|-------|-----------|---------|
| Item Name | flex (remaining space) | left | Item name (semibold, charcoal) + manufacturer below (stone, text-xs) + quantity if set (stone, text-xs, after dot separator) |

**Table sorting (Claude's discretion):** Sort by status pipeline order (ascending: not-yet-ordered first, installed last) using the `STATUS_PRIORITY` map from `ProcurementTable.astro`. This puts actionable items at the top.

**Alternating row stripes:** Even rows: `bg-cream`, odd rows: `bg-cream-dark/50` (matching portal table pattern).

### Inline Status Dropdown (D-01, D-02, D-03)

1. Clicking the status badge opens a positioned dropdown below the badge
2. Dropdown is a `<div>` with `absolute` positioning, `bg-white`, `rounded-lg`, `shadow-lg`, `border border-stone-light/30`, `z-30`, `py-1`, `min-w-[180px]`
3. Each dropdown option: `px-3 py-2 text-sm font-body cursor-pointer hover:bg-cream-dark transition-colors` with the status badge inline
4. Currently active stage: `bg-cream-dark font-semibold`
5. Click outside closes the dropdown (document click listener)
6. `e.stopPropagation()` on the badge container prevents row click from triggering edit
7. On selection: `fetch('/api/admin/update-procurement-status', { method: 'POST', body: JSON.stringify({ projectId, itemKey, status }) })`
8. Optimistic update: local state updates immediately; revert on API error

**Structure:**

**Panel header:** `flex items-center justify-between` with title on left, close button (X icon, 20px, `text-stone hover:text-charcoal`) on right. Bottom border: `border-b border-stone-light/30`. Padding: `px-6 py-4`.

**Panel body:** `overflow-y-auto flex-1 px-6 py-6`. Form fields stack vertically with `gap-4` (16px).

**Panel footer:** `px-6 py-4 border-t border-stone-light/30 flex items-center gap-4`. "Save Item" button left, "Cancel" text link right.

**Backdrop:** `fixed inset-0 bg-black/30 z-40`. Click on backdrop closes the panel.

**Panel z-index:** `z-50` (above backdrop at `z-40`).

### File Upload Zone (D-11, D-12)

**Drop zone appearance:**
- Default: `border-2 border-dashed border-stone-light/30 rounded-lg p-6 text-center cursor-pointer hover:border-stone-light/60 transition-colors`
- Drag-over: `border-terracotta bg-terracotta/5`
- Icon: `Upload` from lucide-react, 24px, `text-stone-light`, centered above text

**Drop zone copy:** "Drop files here or click to browse" in `text-sm text-stone`

**Uploaded file list:** Below the drop zone. Each file as a row:
- Image files: 48px square thumbnail (`rounded object-cover`) + label input + file size + delete button
- Non-image files: `FileText` icon (24px, `text-stone`) + label input + file size + delete button
- Label input: `text-sm font-body border-none bg-transparent text-charcoal placeholder:text-stone-light focus:outline-none` with placeholder "Add label..."
- File size: `text-xs text-stone-light` (e.g., "2.4 MB")

**Upload progress (Claude's discretion):** A thin horizontal bar below the drop zone. Height: 3px. Background: `bg-stone-light/20`. Fill: `bg-terracotta` with `transition-all duration-300`. Shows during upload, disappears on completion after 500ms fade. One progress bar for the current upload; multiple concurrent uploads show cumulative progress.

**Accepted file types:** PDF, JPEG, PNG, WebP (validated server-side; accept attribute: `accept=".pdf,.jpg,.jpeg,.png,.webp"`)

### Overflow Menu (D-09)

**Trigger:** `MoreVertical` icon (lucide-react), 16px, `text-stone hover:text-charcoal cursor-pointer transition-colors`

## Interaction Contracts

### Status Badge Inline Dropdown

1. User clicks the colored badge in the Status column
2. A positioned dropdown appears directly below the badge with all 6 stage options
3. Each option shows the badge in its respective color + the stage label
4. The currently active stage has a `bg-cream-dark` highlight
5. User clicks a new stage
6. Dropdown closes immediately
7. Badge color updates optimistically in local React state
8. `POST /api/admin/update-procurement-status` fires with `{ projectId, itemKey, status }`
9. On API error: revert badge to previous state, show error message
10. Click outside the dropdown closes it without changes
11. `e.stopPropagation()` on the badge prevents the table row from being treated as clickable

### Slide-Out Panel Open/Close

1. User clicks "Add Item" button or "Edit" from overflow menu
2. Backdrop fades in (`opacity 0 -> 0.3` over 300ms)
3. Panel slides in from right (`translate-x-full -> translate-x-0` over 300ms, ease-in-out)
4. For "Edit": form fields pre-populated with existing item data
5. For "Add": form fields empty, status defaults to "not-yet-ordered"
6. User fills form and clicks "Save" / "Add Item"
7. API call fires; button shows "Saving..." disabled state
8. On success: panel closes, table updates with new/edited item, success message appears
9. On error: error message appears in panel, form stays open
10. User can close panel via X button, backdrop click, or Escape key
11. Closing panel without saving discards unsaved changes (no confirmation needed -- form data is not critical enough to warrant save guards)

### File Upload

1. User clicks the drop zone or drags a file over it
2. Drop zone border changes to `border-terracotta bg-terracotta/5` on drag-over
3. On drop/select: progress bar appears below drop zone
4. `POST /api/admin/upload-procurement-file` with FormData (file + projectId + itemKey)
5. On success: file appears in the uploaded file list with thumbnail/icon, label input, size
6. User can type a label in the inline input (saved with the item on "Save Changes")
7. User can delete a file by clicking the trash icon (immediate API call to delete blob + update Sanity)

### Item Removal

1. User clicks three-dot menu on a table row
2. Menu appears with "Edit" and "Remove" options
3. User clicks "Remove"
4. Confirmation dialog appears centered on screen with backdrop
5. Dialog shows: "Remove [Item Name]?" with body text and Cancel/Remove buttons
6. User clicks "Remove": dialog closes, API call fires, item removed from local state optimistically
7. On API error: item re-added to local state, error message shown
8. User clicks "Cancel" or backdrop: dialog closes, no action
