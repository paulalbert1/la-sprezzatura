# Phase 31 -- UI Design Contract

> Visual and interaction contract for the Clients, Contractors, and Portfolio admin sections. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (bespoke Tailwind v4 theme) |
| Preset | not applicable |
| Component library | none (hand-built with Tailwind utility classes) |
| Icon library | lucide-react |
| Font (heading) | Cormorant Garamond, Georgia, serif (`--font-heading`) |
| Font (body) | DM Sans, system-ui, sans-serif (`--font-body`) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, border offsets |
| sm | 8px | Compact element spacing, pill padding, checkbox gaps |
| md | 16px | Default element spacing, form field gaps, card content padding |
| lg | 24px | Section padding, card headers, form section breaks |
| xl | 32px | Layout gaps between cards, main content padding (px-8) |
| 2xl | 48px | Major section breaks, page-level vertical gaps |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Sidebar fixed width: 240px (`w-60`) -- established in AdminLayout
- Contact card popover width: 288px (`w-72`) -- compact card needs specific width
- Table row horizontal padding: 20px (`px-5`) -- established in dashboard cards
- Table row vertical padding: 12px (`py-3`) -- established in dashboard cards

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | DM Sans (`font-body`) |
| Label / Meta | 11px (`text-[11px]`) | 600 (semibold) | 1.4 | DM Sans (`font-body`) |
| Card heading | 13px (`text-[13px]`) | 600 (semibold) | 1.4 | DM Sans (`font-body`) |
| Page heading | 22px (`text-[22px]`) | 600 (semibold) | 1.2 | DM Sans (`font-body`) |

**Exactly 4 sizes:** 11px, 13px, 14px, 22px.
**Exactly 2 weights:** 400 (regular), 600 (semibold).

Notes:
- All admin text uses `font-body` (DM Sans). The serif heading font (`font-heading`) is reserved for the brand name in the sidebar (`Cormorant Garamond`) and the public-facing site only.
- Heading at 22px matches the existing `AdminLayout.astro` pageTitle style.
- `letter-spacing: 0.01em` on card headings (established pattern in dashboard).
- Count badges use `text-[11px]` at weight 600 (`font-semibold`).

## Component Inventory

### 1. Entity List Page (Clients / Contractors)

**Layout:** Full-width white card with search bar at top, table rows below, "New" CTA button in header.

**Focal point:** The table body is the primary scan zone -- the user's eye enters at the top-left name column and scans downward. The "New Client" / "New Contractor" CTA in the top-right provides the primary action anchor.

| Element | Spec |
|---------|------|
| Search input | `text-sm font-body bg-white border border-stone-light/40 rounded-lg px-4 py-2 w-80` placeholder "Search clients..." / "Search contractors..." |
| Search icon | `lucide-react Search`, 16px, `text-stone-light`, positioned inside input left |
| "New Client" button | `bg-terracotta text-white text-sm font-semibold font-body px-4 py-2 rounded-lg hover:bg-terracotta/90` |
| Table header row | `text-[11px] font-semibold text-stone uppercase tracking-wider px-5 py-2 border-b border-stone-light/20 bg-cream/50` |
| Table data row | `text-sm font-body text-charcoal px-5 py-3 border-b border-stone-light/10 hover:bg-cream/50 transition-colors cursor-pointer` |
| Name cell | `font-medium text-charcoal` -- clickable, navigates to detail page |
| Email cell | `text-stone` -- plain text, not a link in the table |
| Phone cell | `text-stone` |
| Preferred contact cell (clients) | `text-[11px] font-semibold px-2 py-0.5 rounded-full bg-stone-100 text-stone-600` badge |
| Trades cell (contractors) | Comma-separated `text-stone`, max 3 visible then "+N more" |
| Empty state | Centered in table body, `text-sm text-stone py-12` |

**Client table columns:** Name, Email, Phone, Preferred Contact
**Contractor table columns:** Name, Email, Phone, Company, Trades

**Sort:** Click column header to toggle ascending/descending. Active sort column header gets `text-charcoal` instead of `text-stone`. Sort indicator: small chevron icon (lucide `ChevronUp`/`ChevronDown`) at 12px beside the column label.

**Pagination:** Not required for initial implementation. Client/contractor datasets are small (<100 records). If list exceeds 50 rows, show a "Showing N of M" count at bottom of table.

### 3. Contact Card Popover

**Trigger:** Hover with 300ms delay on any client/contractor name link. Dismiss on mouse leave with 150ms grace period.

| Element | Spec |
|---------|------|
| Container | `absolute z-50 bg-white rounded-lg shadow-lg border border-stone-light/30 p-4 w-72` |
| Shadow | `shadow-lg` (Tailwind default: `0 10px 15px -3px rgb(0 0 0 / 0.1)`) |
| Name | `text-sm font-semibold text-charcoal font-body` |
| Email | `text-xs text-terracotta font-body hover:underline` -- clickable `mailto:` link |
| Phone | `text-xs text-stone font-body hover:underline` -- clickable `tel:` link |
| Preferred contact | `text-xs text-stone font-body` prefixed with "Prefers: " |
| Divider | `border-t border-stone-light/20 my-2` |
| "View full profile" link | `text-xs text-terracotta font-body hover:underline mt-2 block` |

**Positioning:**
- Default: below and left-aligned to trigger element
- If within 300px of viewport bottom: flip to above
- If within 300px of viewport right: flip to right-aligned
- Use `getBoundingClientRect()` for viewport-aware positioning

**Animation:** `opacity 0 -> 1` over 150ms ease-out on enter; `opacity 1 -> 0` over 100ms ease-in on leave.

### 4. Quick-Assign Typeahead

**Location:** Project detail page, under the clients/contractors section.

| Element | Spec |
|---------|------|
| Input | `text-sm font-body bg-white border border-stone-light/40 rounded-lg px-3 py-2 w-72 focus:ring-1 focus:ring-terracotta` placeholder "Assign a client or contractor..." |
| Dropdown | `absolute z-40 bg-white border border-stone-light/30 rounded-lg shadow-md mt-1 w-72 max-h-48 overflow-y-auto` |
| Dropdown item | `px-3 py-2 text-sm font-body text-charcoal hover:bg-cream cursor-pointer` |
| Item entity type label | `text-[11px] font-semibold uppercase text-stone-light float-right` ("CLIENT" or "CONTRACTOR") |
| No results | `px-3 py-2 text-sm text-stone italic` "No matches found" |
| Selected confirmation | Brief toast-style message: `text-xs text-emerald-600 font-body` "{Name} assigned" for 2 seconds |

**Debounce:** 250ms after last keystroke before filtering.
**Minimum characters:** 1 (dataset is small, immediate filtering is acceptable).

**Contractor follow-up:** When a contractor is selected, a secondary dropdown appears inline below the input asking "Assign for which trade?" showing the contractor's trades list. Same dropdown styling as the typeahead results.

## Form Validation

| Field | Rule | Error Copy |
|-------|------|------------|
| Name (client/contractor) | Required, min 2 characters | "Name is required" |
| Email | Required, valid email format | "Enter a valid email address" |
| Phone | Optional, no format validation | (none) |
| Preferred contact (client) | Required, one of: Email, Phone, Text | "Select a preferred contact method" |
| Company (contractor) | Optional | (none) |
| Trades (contractor) | At least 1 required | "Select at least one trade" |
| Address fields (client) | All optional | (none) |
| Notes (client) | Optional, max 2000 characters | "Notes must be under 2000 characters" |
| Portfolio description | Optional, max 500 characters | "Description must be under 500 characters" |
| Portfolio display title | Optional | (none) |
| Room tags (portfolio) | Optional | (none) |
| Document upload | Max 10MB, PDF/JPEG/PNG only | "File must be PDF, JPEG, or PNG under 10MB" |

Validation is performed client-side for UX and server-side for security. Client-side shows errors inline below the field. Server-side returns `{ error: string }` JSON which is displayed as a toast-style banner at the top of the form.
