## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1`, `gap-1`) |
| sm | 8px | Compact element spacing, row inner gaps (`gap-2`, `py-2`) |
| md | 12px | Row padding, input padding (`px-3`, `py-3`, `gap-3`) -- codebase-inherited exception (not a multiple of 8) |
| lg | 16px | Section heading margins (`mb-4`) |
| xl | 20px | Row horizontal padding (`px-5`) -- codebase-inherited exception from ProjectTasks.tsx (not a multiple of 8) |
| 2xl | 24px | Section spacing between procurement and adjacent sections (`mb-6`) |
| 3xl | 32px | Section outer margin (`mb-8`) -- matches `#procurement` div |

Note: 12px and 20px are codebase-inherited exceptions carried forward from established patterns in ProjectTasks.tsx, ArtifactManager.tsx, and ProcurementTable.astro. They do not conform to a strict 8-point grid but are retained for visual consistency with adjacent sections.

Exceptions: 44px minimum touch target height for status dropdown badges and action icons on mobile. 11px used for timestamps (established pattern from ProjectTasks.tsx, ArtifactManager.tsx).

Source: Established patterns from ProjectTasks.tsx (`px-5 py-3 gap-3`), project detail page (`mb-8`), ProcurementTable.astro (`py-3 pr-3`)

## Component Inventory

### ProcurementEditor (React island)

The main component mounted in `#procurement` div via `client:load`.

**Primary visual focal point:** The status badge is the primary visual focal point of each row. Its color-coded pill draws the eye first and communicates pipeline stage at a glance. All other row content is secondary to the badge.

#### States

| State | Visual treatment |
|-------|-----------------|
| Loading (initial data from Astro props) | Not applicable -- server-rendered initial data, no skeleton |
| Empty (no items) | Centered empty state message with next-step guidance |
| Populated (read mode) | Table rows with status badges, tracking links, pricing columns |
| Row editing (single row) | Inline text inputs, date pickers, cost fields with "Save Changes" / "Discard Changes" buttons |
| Row creating (new item) | Empty editable row appended at bottom |
| Saving | "Save Changes" button shows Loader2 spinner, fields disabled |
| Force-refreshing | Small Loader2 spinner replaces RefreshCw icon on that row |
| Error | Red error banner below table, auto-dismisses after 3 seconds |
| Optimistic update in flight | UI reflects change immediately, reverts on API failure |

#### Layout

```
+----------------------------------------------------------+
| Procurement                              (section heading)|
+----------------------------------------------------------+
| [White card container, rounded-xl, border]               |
|                                                           |
| ITEM   | VENDOR  | STATUS  | EXPECTED | CARRIER | COST  | NET  | TRACK  | ACTIONS |
|        |         |         | DELIVERY |   ETA   |       |      |        |         |
| -----  | ------  | ------  | -------- | ------- | ----- | ---- | ------ | ------- |
| Sofa   | RH      | [badge] | Mar 15   | Mar 14  | $2,500| $800 | UPS    | [icons] |
|        |         | Synced  |          |         |       |      |        |         |
|        |         | 2h ago  |          |         |       |      |        |         |
| -----  | ------  | ------  | -------- | ------- | ----- | ---- | ------ | ------- |
| [+ Add Item row with cream/50 background]                |
+----------------------------------------------------------+
```

Responsive: On screens below `md` breakpoint, hide Vendor, Carrier ETA, Net, and Track columns. Show only Item, Status, Expected Delivery, Cost, Actions.

#### Columns

| Column | Width | Alignment | Visible | Content |
|--------|-------|-----------|---------|---------|
| Item | flex (fill) | left | always | Item name text; red if overdue |
| Vendor | 120px | left | md+ | Vendor name text |
| Status | 112px | center | always | Clickable badge dropdown |
| Expected Delivery | 96px | right | always | Date formatted as "MMM d"; red if overdue |
| Carrier ETA | 96px | right | md+ | Date formatted as "MMM d" or em-dash |
| Cost | 96px | right | always | `formatCurrency(clientCost)` or em-dash |
| Net | 80px | right | md+ | `formatCurrency(retailPrice - clientCost)` or em-dash |
| Track | 72px | center | md+ | Carrier name as link, or truncated tracking # |
| Actions | 64px | center | always | Edit (Pencil), Delete (Trash2), Force-refresh (RefreshCw) |

#### Net price display format (Claude's discretion -- PROC-09)

- Show as plain currency: `$800` (no "Net:" prefix -- the column header already says "Net")
- When negative (clientCost > retailPrice): show `$0` with title tooltip "Client cost exceeds retail price"
- When either value is null/undefined: show em-dash

#### Status dropdown interaction

- Badge is always visible on every row (D-03)
- Clicking the badge opens a dropdown menu below the badge
- Dropdown shows all 6 statuses as selectable options, each with its own badge color
- Current status has a check mark (Check icon from lucide-react)
- Selecting a status auto-saves immediately (D-02) -- no Save button needed
- During save: badge shows brief opacity pulse (50% opacity for 300ms, then back to full)
- Dropdown closes on selection, on click-outside, and on Escape key
- On save failure: revert to previous status, show error banner

#### Sync indicator

- Positioned below the status badge, within the same table cell
- Text: `text-[11px] text-stone-light font-body`
- Content: sync icon (RefreshCw at w-3 h-3) + "Synced 2h ago" (from `formatDistanceToNow`)
- Only visible when `syncSource !== null` AND `lastSyncAt !== null`
- On manual status change: indicator disappears immediately (optimistic)

#### Row edit mode

- Triggered by clicking the Pencil (Edit) icon in Actions column
- Fields become inline inputs: name (text), vendor (text), expectedDeliveryDate (date), installDate (date), retailPrice (number), clientCost (number), trackingNumber (text), notes (textarea -- appears as a full-width row below the main row)
- Number inputs for prices use `type="number"` with `step="0.01"` and display in dollars (convert cents to dollars for display, dollars to cents on save)
- "Save Changes" and "Discard Changes" text buttons appear in the Actions column, replacing Edit/Delete/Refresh icons
- "Save Changes" button: `text-xs font-semibold text-terracotta hover:text-terracotta-light` (text-only)
- "Discard Changes" button: `text-xs text-stone hover:text-charcoal` (text-only)
- Animation: none -- instant swap between read and edit mode (keep it snappy)
- Only one row can be in edit mode at a time; clicking Edit on another row cancels the current edit
- Validation: required fields are name only. Invalid dates or negative prices show `text-xs text-red-600` inline below the field

#### "+ Add Item" row

- Always visible at the bottom of the table
- Background: `bg-cream/50` (matches ProjectTasks.tsx quick-add pattern)
- Left-aligned Plus icon (`w-3.5 h-3.5 text-stone-light`) followed by "Add item..." placeholder text
- Clicking the row enters inline creation mode: empty editable row with same field layout as edit mode
- On successful creation: new row appears at bottom of the item list (above the add row), add row resets
- On validation failure: red inline error text below the name field

#### Force-refresh icon

- Icon: RefreshCw from lucide-react, `w-3.5 h-3.5`
- Default state: `text-stone-light hover:text-terracotta transition-colors`
- Loading state: replaced by Loader2 `w-3.5 h-3.5 animate-spin text-terracotta`
- Only visible on rows that have a tracking number AND status in [ordered, warehouse, in-transit]
- On success: row updates with new data, sync indicator updates
- On failure: show error banner "Could not refresh tracking. Please try again."

#### Action icon accessibility

Each icon-only action button carries an explicit `aria-label` for screen reader support:

| Icon | `aria-label` |
|------|-------------|
| Pencil (Edit) | `aria-label="Edit item"` |
| Trash2 (Delete) | `aria-label="Delete item"` |
| RefreshCw (Force-refresh) | `aria-label="Refresh tracking"` |

#### Delete confirmation

- Reuse the existing `DeleteConfirmDialog.tsx` component
- Extend `entityType` union to include `"procurement-item"`
- Dialog heading: "Delete Item"
- Dialog body: "Remove [item name]? This cannot be undone."
- Cancel button: `text-sm text-stone hover:text-charcoal px-4 py-2 rounded-lg border border-stone-light/40 font-body`
- Delete button: `text-sm font-semibold text-white bg-red-600 hover:bg-red-700 px-4 py-2 rounded-lg font-body`

## Interaction Contracts

### Row-level operations

| Action | Trigger | Visual feedback | API call |
|--------|---------|----------------|----------|
| Edit row | Click Pencil icon | Fields become inputs, "Save Changes" / "Discard Changes" appear | None until Save |
| Save row | Click "Save Changes" text button | Inputs revert to text, spinner on "Save Changes" during request | POST procurement (action: update) |
| Cancel edit | Click "Discard Changes" text button | Inputs revert to previous values | None |
| Change status | Click badge, select from dropdown | Badge updates immediately (optimistic) | POST procurement (action: update-status) |
| Delete row | Click Trash2 icon | Confirmation dialog opens | POST procurement (action: delete) on confirm |
| Force-refresh | Click RefreshCw icon | Spinner replaces icon | POST procurement (action: force-refresh) |
| Add item | Click "+ Add item" row | Empty editable row appears | POST procurement (action: create) on Save |

### Keyboard support

| Key | Context | Action |
|-----|---------|--------|
| Escape | Status dropdown open | Close dropdown |
| Escape | Row in edit mode | Cancel edit (revert) |
| Enter | Add-item name field focused | Submit new item |
| Tab | Within edit-mode fields | Move to next field |

### Optimistic update behavior

All mutations use optimistic UI. On API failure:
1. Revert local state to pre-mutation value
2. Show red error banner below the table: `px-5 py-2 text-xs text-red-600 font-body`
3. Auto-dismiss error after 3000ms (matches ProjectTasks.tsx pattern)
