# Phase 23 — UI Design Contract

> Visual and interaction contract for the custom procurement list item component in Sanity Studio. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Sanity Studio, not a Next.js/Vite frontend) |
| Preset | not applicable |
| Component library | @sanity/ui 3.1.13 (Badge, MenuButton, Menu, MenuItem, Card, Text, Flex, Box, Button) |
| Icon library | @sanity/icons 3.7.4 (EllipsisVerticalIcon, EditIcon, TrashIcon) |
| Font | Sanity Studio default (Inter, system stack) — inherited from Studio theme |

**Note:** This phase builds a Sanity Studio custom array item component. All UI uses @sanity/ui primitives within the Studio environment. There is no standalone frontend build, no Tailwind, and no shadcn. The shadcn initialization gate does not apply.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps between badge and adjacent text |
| sm | 8px | Gap between stacked lines (name/manufacturer, date/tracking) |
| md | 16px | Horizontal padding inside the list row card |
| lg | 24px | Vertical padding on list row card; gap between row sections |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target minimum 32px height for the status badge dropdown trigger and the overflow menu button (Sanity Studio is desktop-only, not mobile).

**Implementation note:** Spacing is applied via @sanity/ui `padding`, `paddingX`, `paddingY`, `space`, and `gap` props which use Sanity's internal spacing scale (0-7). Map: `padding={2}` = 8px, `padding={3}` = 12px, `padding={4}` = 16px. Use `gap={2}` for 8px gaps, `gap={3}` for 12px. Fine-tune with inline `style` props only when @sanity/ui spacing props cannot achieve the required value.

## Typography

All typography uses @sanity/ui `Text` and `Label` components with `size` and `weight` props. Font family is inherited from Sanity Studio theme (Inter).

| Role | @sanity/ui Size | Rendered px | Weight | Line Height |
|------|-----------------|-------------|--------|-------------|
| Item name | `size={2}` | ~16px | `weight="semibold"` (600) | 1.5 (inherited) |
| Manufacturer | `size={1}` | ~13px | `weight="regular"` (400) | 1.5 (inherited) |
| Delivery date | `size={1}` | ~13px | `weight="regular"` (400) | 1.5 (inherited) |
| Tracking number | `size={1}` | ~13px | `weight="regular"` (400) | 1.5 (inherited) |
| Badge label | `size={1}` | ~13px | `weight="semibold"` (600) | 1.2 (badge internal) |
| Menu item text | (default) | ~14px | (default) | (default) |

**Tracking number font:** Rendered with `style={{ fontFamily: 'ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace' }}` on the Text component. Source: D-05.

## Color

This phase operates within the existing Sanity Studio theme established in `studio.css`. No new color tokens are introduced. All colors are consumed from existing sources.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAF5F2` (warm off-white) | List row background (inherits from Studio `--card-bg-color`) |
| Secondary (30%) | `#F0EEEB` (warm gray) | Card hover state, selected item, pane headers (inherits `--card-bg2-color`) |
| Accent (10%) | `#92400E` (amber/gold) | Not used directly in this phase — accent reserved for CTA buttons and active tabs |
| Destructive | Sanity `tone="critical"` | Overdue date text color; Remove menu item tone |

### Status Badge Colors (from procurementStages.ts)

Badges use @sanity/ui `Badge` component with the `tone` prop. Colors are resolved by Sanity's theme engine — do not hardcode hex values.

| Stage | Tone | Visual |
|-------|------|--------|
| Not Yet Ordered | `"default"` | Gray badge |
| Ordered | `"primary"` | Blue/amber badge (per Studio theme) |
| In Transit | `"caution"` | Yellow/amber badge |
| Warehouse | `"caution"` | Yellow/amber badge |
| Delivered | `"positive"` | Green badge |
| Installed | `"positive"` | Green badge |

### Overdue Date Color

- Overdue condition: `expectedDeliveryDate` is in the past AND `status` is not `"delivered"` or `"installed"` (D-09)
- Overdue styling: `<Text tone="critical">` or inline `style={{ color: '#DC2626' }}` (red-600)
- Normal date styling: `<Text muted>` (inherits `--card-muted-fg-color: #64748B`)
- No date set: line omitted entirely (D-10)

### Tracking Number Color

- Default: `style={{ color: '#64748B' }}` (muted gray, matches `--card-muted-fg-color`)
- Hover (when clickable link): `style={{ color: '#475569' }}` (darker gray on hover)
- No tracking number set: line omitted entirely (D-11)

### Dialog Backdrop Overlay

- Value: `rgba(0, 0, 0, 0.4)` applied via `studio.css` on `[data-ui="Dialog"]` (EDIT-01, D-13)

Accent reserved for: CTA buttons (`button[data-tone="primary"][data-ui="Button"]`), active tabs. **Not** used on procurement list items, badges, or menu triggers in this phase.

## Interaction Contracts

### Status Badge Dropdown

1. User clicks the colored `Badge` on a list row
2. A `Menu` appears below the badge with all 6 stage options
3. The currently active stage shows as `pressed={true}` (visually highlighted)
4. User clicks a stage option
5. The status field is patched inline via `inputProps.onChange(PatchEvent.from(set(newStatus, ['status'])))`
6. The badge color updates immediately (React re-render from Sanity form state)
7. The edit dialog does NOT open (click events use `stopPropagation`)
8. The menu closes automatically after selection (MenuButton default behavior)

### Overflow Menu

1. User clicks the three-dot icon (`EllipsisVerticalIcon`) at the far right of the row
2. A `Menu` appears with two items: "Edit" (EditIcon) and "Remove" (TrashIcon, tone="critical")
3. **Edit:** Calls `props.onOpen()` which opens the Sanity edit dialog for this item
4. **Remove:** Calls `props.onRemove()` which triggers Sanity's built-in removal confirmation
5. Click on the overflow icon uses `stopPropagation` to prevent triggering `onOpen`

### Tracking Number Link

1. If `trackingUrl` is set: tracking number renders as an `<a>` element
2. Click opens the URL in a new tab (`target="_blank"`, `rel="noopener noreferrer"`)
3. Hover changes text color from `#64748B` to `#475569`
4. If `trackingUrl` is NOT set: tracking number renders as plain `<Text>` (not clickable)
5. If `trackingNumber` is not set: the tracking line is omitted entirely

### Drag and Drop

1. Drag handle is rendered by `renderDefault(props)` -- no custom drag handle
2. User drags via the native handle to reorder items
3. Order persists when the document is saved
4. No custom DnD implementation needed

### Edit Dialog Overlay

1. User opens the edit dialog (via overflow menu "Edit" or double-clicking the item)
2. A semi-transparent gray overlay (`rgba(0, 0, 0, 0.4)`) appears behind the dialog
3. The overlay is applied via CSS in `studio.css` targeting `[data-ui="Dialog"]`
