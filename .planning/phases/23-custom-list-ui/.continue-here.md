## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Executor worktree stale base | Wave 2 executor was given `bbf700a` as expected base via `worktree_branch_check`, but `git reset --soft <base>` moves HEAD without updating the working tree. The executor committed from a stale working tree, deleting Wave 1 planning files and reverting BlobFileInput/gantt files. Had to restore manually from `bbf700a`. | advisory | After wave merge, verify `git status` is clean before spawning next wave. If executor uses `reset --soft`, follow with `git checkout HEAD -- .` to sync the working tree. |
| @sanity/ui tone colors invisible in warm theme | The Studio's warm beige theme CSS (`[data-ui="MenuButton"] { --card-bg-color: #F0EEEB !important }`) was globally overriding tone-based background colors on ALL MenuButton elements, making status badges look identical (plain gray boxes). Additionally, @sanity/ui `Badge` inside `MenuButton`'s `button` prop does not receive proper tone styling since it's a `<span>`, not a `<Button>`. | advisory | Use explicit `STATUS_PILL` hex color map + `Button mode="bleed"` + inline `backgroundColor`. Scope CSS overrides to `[data-ui="Toolbar"]` only, never globally to `[data-ui="MenuButton"]`. |
