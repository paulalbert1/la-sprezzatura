<objective>
Wire admin authentication into the existing magic-link infrastructure so that the admin email can log in at `/admin/login` and land at `/admin/dashboard` with a valid session, while all other `/admin/*` routes are protected by middleware.

Purpose: Establishes the auth foundation that the admin layout shell (Plan 02) and all future admin phases (26-31) depend on. Without this, no admin page can exist.

Output: Working admin login flow (login -> magic link email -> verify -> session -> dashboard redirect), middleware protection for `/admin/*`, Sanity Studio moved to `/studio`, all tests passing.
</objective>

<threat_model>
- **Privilege escalation via role spoofing:** Mitigated by server-side ADMIN_EMAIL check -- only the env var email gets `role: 'admin'` token. Client/contractor emails cannot trigger admin role.
- **Session fixation:** Mitigated by existing createSession() which generates a fresh random token on every login (never reuses caller-supplied values).
- **User enumeration:** Mitigated by always returning `{ success: true }` from requestMagicLink regardless of whether the email matched ADMIN_EMAIL, client, or nothing.
- **Redirect loop on /admin/login:** Mitigated by adding `/admin/login` to PUBLIC_PATHS array so middleware skips session check for the login page.
- **ADMIN_EMAIL not set:** Handled gracefully -- if env var is missing/empty, the admin detection branch is skipped entirely. No crash, no error. Admin login simply doesn't work (same as if email doesn't match any client).
</threat_model>

if (pathname.startsWith("/admin")) {
  if (PUBLIC_PATHS.some((p) => pathname.startsWith(p))) return next();
