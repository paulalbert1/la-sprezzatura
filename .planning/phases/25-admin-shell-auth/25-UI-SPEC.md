# Phase 25 — UI Design Contract

> Visual and interaction contract for the admin shell, login page, sidebar navigation, top bar, and dashboard landing. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Astro SSR project, no shadcn) |
| Preset | not applicable |
| Component library | none (hand-authored Astro + React islands) |
| Icon library | lucide-react (new dependency, D-08) |
| Font heading | Cormorant Garamond via `--font-heading` (Google Fonts, already loaded) |
| Font body | DM Sans via `--font-body` (Google Fonts, already loaded) |

**Note:** This project is Astro 6 with Tailwind v4 (CSS-first `@theme` tokens in `src/styles/global.css`). No `components.json`, no shadcn, no `tailwind.config.js`. The shadcn initialization gate does not apply.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap in sidebar nav items, brand subtitle margin-top |
| sm | 8px | Compact spacing: nav item vertical padding, between label and input |
| md | 16px | Default element spacing: card internal padding, quick-link gap |
| lg | 24px | Section padding: top bar horizontal padding, dashboard card padding, sidebar section gap |
| xl | 32px | Layout gaps: gap between dashboard summary cards, main content padding |
| 2xl | 48px | Sidebar vertical padding (top/bottom) |
| 3xl | 64px | Not used in this phase |

Exceptions: Sidebar width is fixed at 240px (D-07). This is a layout dimension, not a spacing token.

## Typography

All typography inherits from `src/styles/global.css` base styles. Body line-height is `1.7` (project-wide). Heading line-height is `1.2` (project-wide).

**Declared sizes (4 maximum):**

| Role | Tailwind Class | Rendered Size | Weight | Line Height | Font Family |
|------|---------------|---------------|--------|-------------|-------------|
| Label | `text-xs` | 12px | `font-normal` (400) | 1.5 | DM Sans (`font-body`) |
| Body | `text-sm` | 14px | `font-normal` (400) | 1.7 (inherited) | DM Sans (`font-body`) |
| Heading | `text-xl` | 20px | `font-semibold` (600) | 1.2 | DM Sans (`font-body`) |
| Display | `text-4xl` | 36px | `font-normal` (400) | 1.2 | Cormorant Garamond (`font-heading`) |

**Declared weights (2 maximum):** `font-normal` (400) and `font-semibold` (600). Visual differentiation for the display heading comes from switching to Cormorant Garamond (`font-heading`), which is inherently lighter in feel at weight 400.

### Accent Reserved For (explicit list)

Terracotta (`#C4836A`) is used ONLY on these elements:

1. **Login submit button** -- `bg-terracotta text-white` background
2. **Active sidebar nav indicator** -- left border: `border-l-2 border-terracotta`
3. **Active sidebar nav text** -- `text-terracotta` (active item only)
4. **Dashboard quick-link buttons** -- `bg-terracotta text-white` background
5. **Focus ring** -- `focus:ring-terracotta` on interactive elements (global.css `*:focus-visible`)
6. **Error text** -- `text-terracotta` for validation errors (consistent with portal pattern)
7. **"Try again" link** -- `text-terracotta` in success confirmation state

Accent is NOT used on: sidebar background, top bar, card borders, body text, non-interactive labels.

| Context | Treatment |

| Validation error text | `text-terracotta text-xs` |
| Logout button | No destructive color -- plain `text-stone hover:text-charcoal` text styling |

## Interaction Contracts

### Login Flow

1. User navigates to `/admin/login`
2. Centered form renders with "Welcome back" heading and "Sign in to your studio dashboard" subtitle
3. User enters email and clicks "Send Access Link"
4. Button shows "Sending..." disabled state while request is in-flight
5. On success: form is replaced with "Check Your Email" confirmation (same component state as portal)
6. On rate-limit error: error banner appears with "Too many requests. Please wait a few minutes and try again."
7. On generic error: error banner with "Something went wrong. Please try again or contact liz@lasprezz.com."
8. User clicks magic link in email, lands at `/portal/verify?token=...`
9. Verify page detects `role: 'admin'`, redirects to `/admin/dashboard`

### Middleware Protection

1. Any request to `/admin/*` (except `/admin/login`) triggers session check
2. If no session or `role !== 'admin'`: redirect to `/admin/login`
3. If valid admin session: request proceeds, `Astro.locals.adminEmail` is populated

### Sidebar Navigation

1. All nav items are `<a>` tags (full page navigation, no client-side routing)
2. Active state determined by `Astro.url.pathname.startsWith(href)` comparison
3. Dashboard active when path is exactly `/admin/dashboard` or `/admin`
4. Clicking any nav item navigates to the target page (full reload)
5. Hover state transition: `transition-colors` (150ms default)

### Logout

1. User clicks "Log out" in sidebar (or top bar logout button)
2. Link navigates to `/admin/logout` (API route)
3. API route calls `clearSession(context.cookies)` and redirects to `/admin/login`
4. No confirmation dialog for logout (non-destructive, session can be re-established via magic link)

### Dashboard Quick Links

1. Buttons are `<a>` tags styled as buttons
2. Clicking navigates to target route (will 404 until future phases build those pages)
3. No disabled state needed -- the links exist as forward navigation stubs
