| Token | Value | Usage |

Exceptions:
- Gantt bar height: 28px (Frappe Gantt `--gv-bar-height`, not part of spacing scale)
- Gantt column width: 45px (Frappe Gantt per-day column, not part of spacing scale)
- Smart scroll left-padding: 100px before the target bar (provides visual context before the focused task)
- Fallback scroll left-padding: 50px before the first bar (provides minimal context when scrolling to chart start)

## Visual Behavior Contracts

These are the interaction and rendering rules the executor must implement. Each maps to a requirement.

### GANTT-01: Task Bar Label Visibility

**Contract:** Every task bar rendered by Frappe Gantt must have a visible, readable text label.

| Condition | Expected Behavior |
|-----------|-------------------|
| Bar wider than label text | Label centered inside bar, white fill (`#FFFFFF`) on colored bar background |
| Bar narrower than label text | Frappe adds class `big`, label positioned to right of bar, dark fill (`#0F172A`) on light grid background |
| Milestone / single-day bar (45px) | Label always overflows (class `big`), dark fill, positioned to right |
| Rightmost bar in chart | Label must not be clipped by SVG viewport; ensure SVG width accommodates overflow text |

**Verification:** Playwright snapshot of Gantt chart. Every `.bar-label` SVG element must have a `getBBox()` with `width > 0` and `height > 0`. No label should have computed `fill` matching its background.

### GANTT-02: Scroll to First Task on Load

**Contract:** When the Gantt chart renders, the viewport must show task data, never empty grid space before the first bar.

| Condition | Expected Behavior |
|-----------|-------------------|
| Today is within project date range | Frappe's `scroll_to: "today"` works; chart centers on today. No intervention needed. |
| Today is before project start date | Chart scrolls to the first task bar with 100px left padding for context |
| Today is after project end date | Chart scrolls to the last task bar (or first bar as fallback) |
| `containerRef.current.scrollLeft === 0` after Frappe render (250ms) | Fallback scroll triggers — find first `.bar-wrapper` SVG element, get `getBBox().x`, scroll to `max(0, x - 50)` |

**Timing:** Post-render correction runs at 250ms after Frappe Gantt constructor, piggybacking on the existing `resizeTimer` setTimeout pattern in GanttChart.tsx.

### GANTT-03: Remove Test Page

**Contract:** Delete `src/pages/gantt-test.astro`. No replacement. No redirect.

**Verification:** `test ! -f src/pages/gantt-test.astro` returns exit code 0.

### GANTT-04: Smart Scroll to Nearest Active/Upcoming Task

**Contract:** Extends GANTT-02 with intelligence about which task to scroll to.

| Condition | Scroll Target |
|-----------|---------------|
| Today within range | Frappe handles it (scroll to today column) |
| Today before range: upcoming tasks exist | First task whose `end >= today` (the nearest active or upcoming task), with 100px left padding |
| Today after range: all tasks in past | First task bar (chart start), with 50px left padding |
| No tasks at all | No scroll — empty state shown instead |

**Implementation:** Extract a pure function `computeSmartScrollTarget(tasks, containerEl)` that returns a pixel offset or null. This function is unit-testable.

### LAYOUT-01: Reliable First Pane Hiding

**Contract:** The first structure pane (`[data-pane-index="0"]`) and its following divider must be hidden at all times, including during and after navigation between document types.

| Approach | Details |
|----------|---------|
| CSS-first (prevents flash) | Add `[data-pane-index="0"] { display: none !important; }` and `[data-pane-index="0"] + * { display: none !important; }` to `studio.css` |
| JS backup (MutationObserver) | Replace the 4x `setTimeout` + `setInterval` in StudioNavbar.tsx with a single `MutationObserver` on the structure tool container |
| Observer target | `document.querySelector('[data-testid="structure-tool"]')` with fallback to `document.body` |
| Observer config | `{ childList: true, subtree: true, attributes: true, attributeFilter: ['data-pane-index', 'style', 'hidden'] }` |
| Callback | Debounced `hideFirstPane()` (reuse existing function) |
| Cleanup | `observer.disconnect()` in useEffect return |

**Verification:** Playwright snapshot after navigating between Projects, Clients, and Contractors tabs. `[data-pane-index="0"]` must have `display: none` in each snapshot. No visible flash of the Content type list.
