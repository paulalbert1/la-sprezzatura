## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, tier card line-item rows |
| md | 16px | Default element spacing, form field gaps, tier card internal spacing |
| lg | 24px | Card internal padding (p-6), section heading to content gap, email section spacing |
| xl | 32px | Layout gaps, page horizontal padding, email CTA vertical margin |
| 2xl | 48px | Major section breaks (py-12), email content card padding |
| 3xl | 64px | Page-level spacing (mt-16 on sign-out links) |

Exceptions:
- 12px (`space-y-3`, `pb-3`, `gap-3`, `mb-3`): Compact list item spacing for tier line items and milestone rows. Established codebase pattern (MilestoneSection, ProcurementTable). Multiples-of-4 compliant (12 = 3 x 4).
- 80px (`md:py-20`): Desktop page-level vertical padding on portal main content. Established pattern from Phase 5 onward. Multiples-of-4 compliant (80 = 20 x 4).
- 44px minimum touch target height on all portal buttons (Select This Tier, Confirm Selection) and email CTA -- WCAG 2.2 Target Size.
- 40px (`py-10`): Email content card top/bottom padding. Matches existing `notify-artifact.ts` email template exactly (`padding: 40px 32px`).
- Sanity Studio dialog components use `@sanity/ui` spacing tokens, not Tailwind tokens.

## Typography

Portal / web typography (4 tiers):

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Label | 12px (`text-xs`) | 400 (regular) | 1.5 | DM Sans | Uppercase tracking-widest labels, help text, tier descriptions, readiness labels, email section headings, tier line item names, badge text, timestamps |
| Body | 16px (`text-base`) | 400 (regular) | 1.7 | DM Sans | Paragraph text, personal note textarea, reservation textarea, form body copy |
| Heading | 20px (`text-xl`) | 300 (light) | 1.2 | Cormorant Garamond | Section headings (Investment Summary, Readiness Check) |
| Display | 40px mobile / 48px desktop (`text-4xl md:text-5xl`) | 300 (light) | 1.2 | Cormorant Garamond | Tier names on desktop, hero title (responsive: `text-6xl md:text-7xl lg:text-8xl xl:text-9xl`, `leading-none`) |

Weights used: 300 (light, headings/display) and 400 (normal, body/label). Two weights only. Emphasis achieved through `text-charcoal` color (against `text-stone` for secondary text) and uppercase tracking-widest for labels.

Note: Hero Display uses the Display tier at responsive sizes (`text-6xl` through `text-9xl`) with `leading-none` -- this is a responsive variant of Display, not a separate tier.

## Sanity Studio: SendUpdateAction Dialog

### Visual Specification

Uses `@sanity/ui` primitives and inline styles (established pattern from `notifyClient.tsx`, `sendWorkOrderAccess.tsx`, `sendBuildingAccess.tsx`). Appears on project documents only.

**Action button (in Studio toolbar):**
- Label: "Send Update"
- Tone: "primary"

### Dialog Layout

```
Dialog (header: "Send Project Update")
  Stack (space: 4)

    [Frequency Warning -- conditional]
      Card (tone: "caution", padding: 3)
        Text (size: 1):
          "Last update sent {X} hours ago. Send another?"
      Visible only when last updateLog entry sentAt < 24 hours ago.

    [Personal Note Section]
      Label: "Personal Note"
        TextArea (rows: 3)
          Default value: "Hi {PrimaryClientName}, here's the latest on {ProjectTitle}."
          Placeholder: "Add a personal note..."

    [Section Toggles]
      Label: "Include Sections"
      Stack (space: 2)
        Checkbox: "Milestones" -- default checked
        Checkbox: "Procurement" -- default checked, hidden if engagementType !== "full-interior-design"
        Checkbox: "Pending Artifact Reviews" -- default checked

    [Preview Section]
      Label: "Preview"
      Card (padding: 3, border: true)
        Simplified preview rendering:
          [Personal note text -- if not empty]
          [Milestones heading + count -- if toggled on]
            "{N} milestones ({M} complete)"
          [Procurement heading + count -- if toggled on]
            "{N} procurement items"
          [Artifacts heading + count -- if toggled on]
            "{N} pending reviews"
          [View in Your Portal -- terracotta text]

    [Send Error Message -- conditional, shown on API failure]
      Card (tone: "critical", padding: 3)
        Text (size: 1):
          "Unable to send update. Please check your connection and try again."

    [Action Buttons]
      Flex (gap: 2)
        Button (tone: "primary", text: "Send Update")
          idle: "Send Update" (bg: #C4836A via tone)
          sending: "Sending..." (disabled)
          success: dialog auto-closes, props.onComplete() called
          error: re-enables button for retry, shows error card above
        Button (tone: "default", text: "Discard", mode: "ghost")
          onClick: close dialog
```

### Preview Rationale

The preview shows a simplified summary (section headings + data counts) rather than full HTML email rendering. Rationale: full HTML rendering inside a Sanity Studio dialog requires an iframe or dangerouslySetInnerHTML with significant complexity for marginal benefit. The preview confirms which sections are included and that data exists. The actual email uses the proven branded template.

### Email Section Visibility Rules

| Section | Visible When |
|---------|-------------|
| Personal note | Always (pre-filled, Liz can edit or clear) |
| Milestones | Toggle ON (default) AND project has milestones |
| Procurement | Toggle ON (default) AND engagementType === "full-interior-design" AND project has procurement items |
| Pending Reviews | Toggle ON (default) AND project has artifacts awaiting review |

If ALL data sections are empty (no milestones, no procurement, no pending reviews), the email contains only the personal note and CTA button. The toggles still show but sections render as empty. This is acceptable -- Liz may want to send just a personal note.

## Hero Slideshow + SplitText Animation

### Visual Specification

Full rewrite of `src/components/home/Hero.astro` per the authoritative spec at `docs/superpowers/specs/2026-03-16-hero-slideshow-design.md`. This section documents the UI contract; the spec document is the implementation reference.

### Layout

```

  [Slide container -- all slides stacked absolutely]

      Each slide:
        div.hero-slide.absolute.inset-0.opacity-0.transition-opacity.duration-[1500ms].ease-in-out

    div.absolute.inset-0
      background: linear-gradient(to top, rgba(44,41,38,0.65) 0%, rgba(44,41,38,0.2) 50%, rgba(44,41,38,0.05) 100%)

    div.absolute.bottom-0.left-0.right-0.px-8.md:px-16.lg:px-24.pb-20.md:pb-28
      [Tagline -- renders ABOVE title, fades in after SplitText completes]
        p.text-xs.uppercase.tracking-[0.35em].font-body.mb-5
          color: rgba(184,176,164,0.65)
          opacity: 0 initially, fades to 1 after title animation completes
          "{tagline}"

      [Title -- SplitText target]
        h1.hero-title.font-heading.text-6xl.md:text-7xl.lg:text-8xl.xl:text-9xl.font-light.leading-none.tracking-wide
          color: rgba(255,255,255,0.65)
          "{title}" (default: "La Sprezzatura")

  [Scroll indicator -- line only, no text]
    div.absolute.bottom-6.left-1/2.-translate-x-1/2 [aria-hidden="true"]
      div.w-px.h-9.scroll-line
        color: rgba(255,255,255,0.4)
        animation: slow downward draw, 3s cycle, scaleY transform-origin top

  [Progress bar -- very bottom edge]
    div.absolute.bottom-0.left-0.right-0.h-0.5
      [Track]: background rgba(255,255,255,0.1), full width
      [Fill]: background rgba(196,131,106,0.4), width animates 0% to 100% per slide
```

### SplitText Animation Contract

| Property | Value |
|----------|-------|
| Target element | `.hero-title` (the h1) |
| Split type | `chars` (character-level) |
| Initial state | Each character: `opacity: 0`, `y: 12px` |
| Animation | `opacity: 1`, `y: 0` |
| Duration per character | 60ms |
| Stagger | 30ms between characters |
| Easing | `power2.out` |
| Total approximate duration | ~0.8s for "La Sprezzatura" (14 chars: 0.06 + 14 x 0.03 = 0.48s visible portion) |
| Tagline fade-in | Starts after SplitText timeline completes, 400ms duration, `power2.out` |
| Plays | Once on `astro:page-load`, does NOT replay on slide transitions |
| Cleanup | `splitInstance.revert()` on `astro:before-swap` (view transition safe) |
| Reduced motion | If `prefers-reduced-motion: reduce`, skip SplitText animation entirely (title and tagline render at full opacity immediately) |

### Ken Burns Contract

| Property | Value |
|----------|-------|
| Scale range | 1.0 to 1.06 (desktop), 1.0 to 1.03 (mobile) |
| Direction cycle | push-in, drift-left, pull-back, drift-right (repeats via `i % 4`) |
| Duration | Full 8.5s per slide (hold + fade) |
| All directions | Scale outward from 1.0 -- never shrink below 1.0 |
| Reduced motion | Disabled; images remain static |

### Slideshow Timing Contract

| Property | Value |
|----------|-------|
| Crossfade type | Pure dissolve (simultaneous opacity transition) |
| Crossfade duration | 1.5s, ease-in-out (CSS transition) |
| Hold duration | 7s per image |
| Total cycle per slide | ~8.5s |
| Looping | Infinite, wraps from last slide to first |
| Image readiness | Check `img.complete === true` before advancing; hold current slide if target not loaded |
| Reduced motion | Auto-advance paused, shows first image only, progress bar hidden |

### Fallback Chain

```
heroSlideshow (2+ images)  -->  slideshow with crossfade + Ken Burns + SplitText
heroSlideshow (1 image)    -->  single static image + SplitText (current behavior with animation)
heroSlideshow (empty/null) -->  gradient fallback (current behavior)
```

## Interaction Contract

### Send Update Flow

```
[Liz clicks "Send Update" on project in Studio]
  --> [Dialog opens]
    --> [Pre-filled note, all toggles ON]
    --> [If last update < 24h: frequency warning shown]
    --> [Liz edits note, toggles sections, sees preview update live]
    --> [Click "Send Update"]
      --> [Sending... state, button disabled]
        --> [POST /api/send-update with projectId, note, sections]
          --> [API: fetch project data via GROQ]
          --> [API: build branded HTML email per template]
          --> [API: send via Resend to all project clients]
          --> [API: log updateLog entry on project document]
        --> [Success: dialog auto-closes, props.onComplete()]
        --> [Error: show "Unable to send update. Please check your connection and try again." in Card (tone: "critical") above action buttons, re-enable Send Update button for retry]
    --> [Click "Discard"]
      --> [Dialog closes, no action taken]
```

### Tier Selection Flow

```
[Client views proposal artifact on project detail page]
  --> [InvestmentSummary renders tier cards side by side]
  --> [Client clicks "Select This Tier" on preferred tier]
    --> [Other tier cards fade (opacity-50)]
    --> [Selected tier card shows readiness check inline]
      --> [Client selects eagerness (1-5 circle scale)]
      --> [Client optionally enters reservations]
      --> [Client checks confirmation checkbox]
      --> [Client clicks "Confirm Selection"]
        --> [Submitting... state]
          --> [Astro Action: selectTier]
            --> [Write selectedTierKey, eagerness, reservations to artifact]
            --> [Add decision log entry with timestamp]
          --> [Success: "Selection recorded", page reloads after 2s]
          --> [Error: error banner with retry]
      --> [Client clicks "Go Back"]
        --> [Returns to idle state, all tiers at full opacity]
```

### Hero Slideshow Flow

```
[Page load (astro:page-load)]
  --> [Check prefers-reduced-motion]
    --reduce--> [Show first image static, title/tagline at full opacity, no animations]
    --no-preference-->
      --> [Set slide 0 to opacity-1]
      --> [Start Ken Burns on slide 0]
      --> [Initialize SplitText on .hero-title]
      --> [Animate characters in (stagger)]
      --> [On SplitText complete: fade in tagline]
      --> [Start slideshow timer (7s hold)]
        --> [On timer: check next slide img.complete]
          --loaded--> [Crossfade to next slide, start Ken Burns on new slide, reset progress bar]
          --not loaded--> [Hold current slide, retry on next tick]
        --> [Loop: last slide wraps to first]

[Navigation away (astro:before-swap)]
  --> [Clear slideshow interval]
  --> [splitInstance.revert()]
  --> [Reset all state]
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Focus visible | `outline: 2px solid var(--color-terracotta); outline-offset: 3px` (global.css) |
| Touch targets | Minimum 44px height on all interactive elements (Select This Tier, Confirm Selection, eagerness circles at 32px with 44px tap area via padding) |
| Form errors | `role="alert"` on error messages |
| Form labels | All inputs have associated `<label>` elements |
| Color contrast | Charcoal on cream = 12.5:1 (AAA). Stone on cream = 3.6:1 (AA large). Terracotta on white = 4.5:1 (AA). Hero title white@65% on dark overlay -- verify >= 4.5:1. |
| Eagerness scale | `role="radiogroup"` with `role="radio"` children, `aria-checked`, `aria-label` on each ("1 - Just exploring", etc.) |
| Keyboard nav | Tab order follows visual order. Eagerness circles: arrow keys navigate, Space/Enter selects. All interactive elements reachable via keyboard. |
| Reduced motion | `prefers-reduced-motion: reduce` disables Ken Burns, SplitText, slideshow auto-advance. Title/tagline render at full opacity immediately. Progress bar hidden. |
| SplitText a11y | SplitText sets `aria-label` on the parent element preserving the full text for screen readers while characters are split into individual spans. |
| Slideshow images | All slideshow images require alt text (Sanity schema validation `Rule.required()` on alt field) |
| Email a11y | Email uses semantic HTML: `<h1>`, `<h2>`, `<table>` with proper structure. Alt text on any images. |
| Scroll indicator | `aria-hidden="true"` -- decorative element |
| noindex/nofollow | All portal pages include `<meta name="robots" content="noindex, nofollow">` via PortalLayout.astro |

## Responsive Behavior

| Breakpoint | Layout Changes |
|------------|---------------|
| Mobile (<768px) | Tier cards stacked vertically (grid-cols-1). Hero text: text-6xl. Ken Burns scale max 1.03. Slideshow timing unchanged. Readiness check form full-width. |
| Tablet (768px-1023px) | Tier cards: 2-column grid if 2+ tiers. Hero text: text-7xl. Ken Burns scale max 1.06. |
| Desktop (>=1024px) | Tier cards: up to 3-column grid (3 tiers), 2-column for 4+ tiers. Hero text: text-8xl (lg), text-9xl (xl). |

Email: Fixed 600px max-width, mobile email clients render single-column naturally. Table-based layout ensures compatibility across Outlook, Gmail, Apple Mail.
